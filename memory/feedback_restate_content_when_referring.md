# ルール

複数の選択肢に **A / B / C / D** や **B1 / E1** 等の **短縮 ID を割り当てて議論** している時、 後続の応答で参照する場合は **必ず本文を併記** すること。

NG 例: 「D 採用、 B1 と E1 は次点」
OK 例: 「**D**「むずかしかったのう。 また やってみるとよい」 採用、 **B1**「かんがえたのが えらいぞ」 と **E1**「ひとやすみして また おいで」 は次点」

## Why
ユーザー (2026-05-12 明示フィードバック): 「D って何？毎回、 表現がわからないんだ。 前の点灯 [変換ミス: 候補] を引っ張り出してくるって、 こっちが面倒なんだよ。 スクロールしてどこまで打てばいいかなんて、 そういう時は D をちゃんと出してよ、 一瞬で」

過去のセッションで **何度も発生している恒久問題** のため、 一回限りの注意ではなく恒久ルールとして登録。

## How to apply
- 1 メッセージ内で初出時に ID + 本文 を一度書いた後、 同じメッセージ内なら ID 単独 OK
- **次のメッセージ以降で参照する時は必ず本文を併記** (前メッセージの内容をユーザーが覚えているとは想定しない)
- 候補リストが 3 件以上ある時、 4 件以上ある時、 など件数を問わず一律適用
- 表で提示する場合も「ID 列」 と「本文列」 を必ず両方含める
- 略称・ID は **スペース節約のためでなく、 議論を分かりやすくするため** に使う。 本文を省くと逆効果

## 適用範囲
このプロジェクト (pono-asobiba-app) のみならず、 **全プロジェクト共通**。 ユーザーの認知負荷を下げるための一般原則。
