## ★アセットシート方式 (重要 / 必須 — 2026-05-06)

**ペアの 2 枚を別々に生成しないでください**。 1 ペア = 1 枚のアセットシートとして生成:

- **1 つのプロンプトで 1 枚の画像** を出力
- その 1 枚を**横並び 2 パネル** (left = img_word / right = img_answer) にする
- パネル間にうっすら境界線 or 余白を入れる (ユーザーがクロップしやすいように)
- **カメラ画角・ライティング・小道具・キャラ衣装** はピクセル単位で全パネル共通
- 違うのは**対比属性のみ** (位置・動作・表情・状態)

**理由**: 別々生成だとカメラ角度・小道具細部・服の柄が微妙にズレ、 子どもの注意がペアの主旨ではなくズレに行ってしまう。 1 枚で生成すれば構図/素材が強制的に揃う。

**Codex 側のワークフロー (2026-05-06b)**:
1. シートを 1 枚生成
2. **alpha 抜き + 切り抜き**を Codex 側で実施 (左パネル / 右パネルに分割、 個別 PNG 化)
3. クロスレビューエージェントが視覚確認 (style 一致、 alpha 漏れなし、 crop ズレなし、 解像度 ≥1024)
4. NG 判定なら再生成 → 再 review
5. 合格した個別 PNG を `tmp/alpha_pending/alpha/05/` に保存

### 解像度の最低条件 + フォールバック

- アセットシート方式は、 **シートを切り出した後、 各パネルが必要解像度以上 (≥1024×1024 相当)** であることが最低条件。 横並び 2 パネル = シート全体で **≥2048×1024 相当**が必要。
- 生成 API のネイティブ出力サイズで上記を満たせない場合は**別々生成にフォールバック OK**。 ただしその際は:
  - **1 つ目を生成 → その画像をリファレンスとして 2 つ目を生成**する (image-to-image 連鎖)
  - 2 枚目が 1 枚目の構図・小道具・キャラデザを引き継ぐので、 別々でもアセットシート相当の整合性になる
- **絶対 NG**: リファレンス無しの完全独立生成 2 回 (= 従来やっていたパターン、 これを止めるためのこの方針)

**ポリシー履歴**:
- 〜2026-05-05: Codex 側で alpha 抜き → ばらつきあり
- 2026-05-06a: ユーザー手作業 (品質安定優先で一時撤退)
- **2026-05-06b 現行**: チーム編成 + クロスレビュー体制で再び Codex 側で alpha 抜き + crop を実施 (実験中、 ダメなら手作業に戻る)

## 12. Q140「うえ」/「した」 — 2 枚

### 出力ファイル
- `ue.png` (img_word)
- `shita.png` (img_answer)

### コンセプト
**棚 (or テーブル) の段差 + りんご** で「上にある / 下にある」を位置関係で示す。 「たかい/ひくい」(物自体の高さ — 既に Q131 takai_hikui.png で対応済) と区別するため、 **同じ位置に置かれた棚の上下段でりんごの位置だけ変える**。

### 共通シーン
- **木製の 2 段棚** (or 横長の 2 段テーブル)。 シンプルなナチュラルウッド、 上段と下段の高さ差が明確
- 棚は画面中央、 ue/shita で**完全に同じ棚**
- りんごは赤いりんご 1 個 (既存 ringo 系と同じ画風)
- 透過背景、 棚の足元に薄影

### ue.png (うえ)
- 棚の**上段にりんごが 1 個**置かれている
- 下段は空 (何も置かれていない)
- 矢印やラベルは不要、 位置だけで伝える

### shita.png (した)
- 棚の**下段にりんごが 1 個**置かれている
- 上段は空
- カメラアングル・棚のサイズ・りんごの種類すべて ue.png と同一

### 注意
- 「たかい/ひくい」と被らないように **棚自体の高さは変えない** (棚は同じ、 りんごの位置だけ違う)
- りんご 1 個ずつ (複数だと「数」と混同)
