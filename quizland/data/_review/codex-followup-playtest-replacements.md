## 命名規則と納品方法

- ファイル名は本ドキュメント内の指定通り (snake_case、 stage プレフィックス、 opposite/ サブフォルダは opposite カテゴリ専用)
- alpha 抜き済 RGBA PNG として納品
- `tmp/alpha_pending/alpha/07/` にまとめて投入 → user が手動で alpha 仕上げ後 assets/ に配置
- 各納品物は別エージェントが visually review、 アルファ漏れ・構図ずれ・グロ表現・ネタバレ要素のチェック必須

## 検収観点

1. 答えのネタバレ要素が img_word に入ってないか (例: 雨地面にミミズ NG)
2. 5 歳児が見て怖くないか・グロくないか (傷・骨・歯のリアル過剰禁止)
3. 既存ステージング画像とのスタイル統一 (水彩 + 鉛筆線 + パステル)
4. 透過背景が完全に抜けてるか (端のフリンジ要確認)
