# ひょっこりハイタッチ「もりの さんぽみち」企画

作成日: 2026-07-23

対象: `hyokkori-hightouch/` / `assets/images/hyokkori-hightouch/`

状態: 2026-07-24に固定ルート5地点と物語の一本線まで実装済み。こもれび＝黄緑、どんぐり＝赤茶・琥珀、みずべ＝青緑・水色、きのこ＝珊瑚・すみれ、つきあかり＝群青・銀へ色脚本を分離し、全地点を画像中心ではなく接地点基準で配置する。旧3地点完走セーブは4面から再開し、5面完走後は月の花が咲いて1面へ周回する。自由選択と場所固有の環境反応は未実装。

## 1. 結論

「ひょっこりハイタッチ」を、同じ森で点数を繰り返すゲームから、**どうぶつたちとハイタッチして、ひかりの たねを つきの はなまで届ける短い森の旅**へ広げる。

- 1ゲームは現状どおり30秒で完結する。
- 30秒を最後まで遊ぶと、点数に関係なく必ず次の場所へ1歩進む。
- 場所ごとに背景、かくれ場所の数・配置・外装、会える動物を変える。
- プレイ中のルールと表示は増やさない。変化は景色と顔ぶれで伝える。
- `ひかりの たね`はプレイ中の育成ゲージへ戻さない。開始前の一言、結果の散歩道、5面終了後の開花だけで目的を伝える。
- 第1弾の3場所で場所替えを検証した後、第2弾として夕方と月夜の2場所を追加し、5場所の旅を完成させた。
- 5場所完成後も一周を強制しない。各30秒の終了時に保存し、いつでも中断・再開できる。

この案の再プレイ動機は、低確率の報酬ではなく、**「次に行けば、次の景色と動物に必ず会え、種を目的地へ必ず近づけられる」**ことに置く。

規模の目安は、現在の1場所をデータ化するPhase 0がM、3場所の画像と散歩進行まで含めたPhase 1がM+。5場所の固定ルート完成と、自由選択・場所固有反応は分けて実装する。

### 物語の一本線

1. 朝の`こもれびの ひろば`で、ひかりモモンガが小さな`ひかりの たね`を見つける。
2. 起きているどうぶつとのハイタッチで光を分けてもらい、種を森の奥へ運ぶ。
3. `どんぐりの こみち`で道を教わり、`せせらぎの みずべ`でかわうそに対岸へ渡してもらう。
4. 夕方の`きのこの おか`では、きのこの灯りを目印に坂を登る。
5. 夜の`つきあかりの もり`へ種を届けると、結果画面で`つきの はな`が咲く。
6. 花から新しい種が一粒飛び、次の散歩へ自然につながる。

点数やコンボで花の光り方は少し変えてよいが、低得点でも30秒完走すれば必ず物語は進む。会話画面、持ち物欄、プレイ中の種メーターは追加しない。

## 2. ゲームの核

### 変えないもの

| 項目 | 方針 |
| --- | --- |
| 主操作 | おきている子へハイタッチ |
| 予備判断 | ねている子はタップせず、そっと待つ |
| 1ゲーム | 30秒 |
| 得点 | 通常10点、高得点のひかりモモンガ30点、コンボ加点 |
| コンボ | 中央の大きな数字と段階的な花火、最大コンボ記録 |
| 出現 | 最大同時2体、7体ごとにひかりモモンガ |
| 難易度カーブ | 出現間隔、表示時間、ねむり比率を現状維持 |
| 失敗の扱い | 減点や道の後退なし。取り逃しても進行を失わない |

### 場所ごとに変えるもの

| 項目 | 変化 |
| --- | --- |
| 背景 | 明るさ、植生、地形、時間帯 |
| かくれ場所 | 4〜6か所。場所内では同じ外装に統一 |
| 配置 | 3＋3、3＋2、2＋2、2＋3など、視線の巡り方を変える |
| 動物 | 各場所6種類。既存動物を再編し、新規は4種類だけ追加 |
| 成功後の反応 | 木漏れ日、どんぐり、波紋など一瞬の環境反応。正解ヒントには使わない |

場所が進んでも「後半ほど速い」「穴が多いほど難しい」という階段にはしない。対象サイズ、表示時間、同時出現数、得点機会は揃え、**難易度差ではなく探し方と景色の差**にする。

## 3. 基本の体験フロー

1. 開始画面で今回の場所と、散歩道の現在位置を短く見せる。
2. `ここで あそぶ`で開始する。
3. カウントダウン中だけ場所名を約1秒表示し、プレイ開始時には消す。
4. 30秒間、現状と同じハイタッチを遊ぶ。
5. 結果で点数・最大コンボを表示する。
6. ポノの足あとが次の場所へ1マス進む。
7. `さんぽを つづける`で次の背景を読み、場所名付きカウントダウンへ直行する。タイトル画面へは戻さない。

開始画面を見せるのはゲームへ入った最初の1回だけ。結果画面の主ボタンは`さんぽを つづける`の1つに絞り、終了は既存のホーム導線を使う。

### 進行条件

- 0点でも、30秒を最後まで遊べば進む。
- ねている子を誤タップしても後退しない。
- ひかりモモンガを取り逃しても進む。
- タブを閉じる、ホームへ戻るなど30秒未完了の時は進めず、同じ場所から再開する。
- 30秒終了直後に進行を一度確定する。結果処理やボタンが二重に呼ばれても1ゲームで2マス進まない。

太字の4種類だけが新規動物。既存8種類を場所ごとに組み替えることで、画像点数を抑えながら顔ぶれの変化を明確にする。第1弾前半は`たぬき`と`かわうそ`の2種類だけを新規作成し、みずべの`かえる`枠には一時的に`ふくろう`を入れる。`やまね`と`かえる`は第2弾で追加する。

ひかりモモンガは全場所共通の「散歩の案内役」とする。場所固有の高得点キャラは増やさず、7体ごとの30点、見分けやすい光、長めの表示時間を維持する。

### 色脚本

場所の違いを地形だけに頼らず、主役色と光源色でも伝える。現行3地点は、こもれびのレモン黄緑を基準として残し、どんぐりは赤茶／琥珀、みずべはターコイズ／青灰へ分離する。後半は、きのこを珊瑚／すみれ紫、つきあかりを群青／銀へ進める。

- 隣接地点の全体平均色は、目安として`ΔE76 >= 20`を確保する。
- 中央コンボ安全帯も全地点で同じ黄クリームにせず、各地点の淡い主役色にする。
- 既存背景を生成参照にする場合は、筆致・カメラ・安全帯だけを継承し、色と光源色は継承しない。
- 黒や濃紺を大面積で使わず、各出現位置の背後は動物の顔・睡眠月・ボーナス光を読める中明度以上に保つ。

Phase 1の3場所は、背景ごとの地面へなじむfar／near外装を各1点ずつ使う。別の場所の外装や共通の葉の茂みを流用せず、接地点と前縁マスクも6点それぞれで計測する。

高速表示でも混同しにくいよう、`たぬき`と`あらいぐま`、`やまね`と`りす`は同じ場所へ入れない。新規画像では顔模様、尾、耳、全身シルエットの差を生成要件にする。

## 5. かくれ場所の配置

座標は16:9のゲームステージ左上を `(0, 0)` とした、**外装の地面接地点**の割合。画像キャンバス中心ではない。実装時は固定ピクセルではなく正規化した`x / groundY`として持ち、素材ごとの`groundAnchorY`を同じ点へ合わせる。

| 場所 | 奥／中景 | 手前 |
| --- | --- | --- |
| こもれびの ひろば | `(27,52)` `(78,49)` `(24,70.5)` `(76,70)` | `(18,91.5)` `(82,91.5)` |
| どんぐりの こみち | `(18,52)` `(82,38)` | `(27,81)` `(72,77)` `(50,94)` |
| せせらぎの みずべ | `(13,47)` `(80,45)` | `(20,88)` `(80,87)` |
| きのこの おか | `(30,55)` `(71,40)` | `(18,78)` `(50,84)` `(82,77)` |
| つきあかりの もり | `(27,39)` `(73,39)` | `(21,79)` `(79,79)` |

### 共通の安全条件

- 中央コンボ安全帯の構図基準をおおむね `x=42〜58% / y=43〜66%` として空ける。合否は固定率だけで決めず、最大tierの数字・glow・花火の実測bboxに12px以上の余白を加えて判定する。
- 奥の上段は見た目だけ約0.88倍、手前の下段は1.0倍。操作面は縮小しない。
- タップ領域は最低44px、目標56px以上を確保する。
- 568×320でもHUD、中央コンボ、別のかくれ場所と重ならない。
- 各スロットは中心アンカーの`slotBox`を持ち、目安を`width: clamp(84px, 22cqw, 260px)`、`height: clamp(72px, 30cqh, 190px)`とする。外装はslot幅の約92%、キャラ窓は約58%から調整する。
- DOM順は見た目の左上から右下へ並べ、キーボード移動と読み上げ順を一致させる。
- 各スロットは実button、または同等のkeyboard／支援技術操作を持たせる。Enter／Spaceはpointerと同じ1回の判定経路へ入り、touch後の合成clickとの二重発火を防ぐ。
- 4か所の場所は探す候補が少ない代わりに対象を小さくしたり速くしたりしない。
- 6か所の場所でも同時出現は最大2体のままにする。

## 6. 背景とマスクの作り方

今回の位置ずれ、出現途中の下漏れ、停止時の下端クロップを別の場所で再発させないため、背景とかくれ場所を次の構造に分ける。

1. **背景**

   地形、草、道、水面などの広い景色だけを描く。厳密な穴影や茂み本体は焼き込まない。かくれ場所を置く範囲には、広めの自然な接地面だけを用意する。

2. **かくれ場所の外装**

   場所ごとに遠景用と近景用の2種類を用意する。遠景用は低く平たく、近景用は少し立体的に描くが、どちらも同じ操作・同じ得点の「かくれ場所」とする。違いはカメラからの距離だけに限定し、形によって効果が変わるようには見せない。

3. **キャラ窓**

   全外装で開口中心、停止位置、下端マスクを共通規格にする。

4. **接地点stackと手前マスク**

   `hh-ground-stack`の上辺中央を背景上の`groundY`へ置き、ベース、キャラ窓、手前縁、空振りFXを必ず一緒に移動・拡縮する。外装画像はalpha実体下端から求めた`groundAnchorY`だけstack内で上へ戻す。scaleとpressed演出の`transform-origin`も接地点へ置き、画像中心回転で片側を持ち上げない。

新しい背景は16:9を基準にし、4〜6か所の構図候補を含む広い接地帯を描く。cover-safe領域、HUD、中央コンボ帯を画像briefへ明記し、背景単体ではなく予定座標へ外装を重ねた合成プレビューを全viewportで承認する。実装用は1600×900へ最適化する。

新しい外装は1254×1254 RGBAの正方形、動物は既存と同じ1200×1200 RGBAの正方形を基準にする。生成後に、四隅alpha 0、半透明フリンジ、開口中心、前縁開始位置、停止位置、awake／sleepingの足元と外接bbox、耳・翼・尾を機械検査と実画面の両方で確認する。

背景検収では、穴を置く予定位置だけでなく画面全域を拡大確認する。透明ドーム、泡、ガラス状アーチ、石の円環、独立した台座、背景に焼き込まれた穴らしき形は生成不良として不採用にする。水辺は接地影ではなく、連続した湿った土と非対称な少数の石で岸へなじませる。

### 2026-07-24 アート方向の補足

3地点の差は、共通背景へ強い色調補正を重ねて作らない。紙の風合いが残る水彩・色鉛筆調を共通の画風とし、写真的な材質感、細密な毛・石・水面、強い被写界深度は避ける。

- こもれびの ひろば: 子どもの目線に近い低いカメラ。手前の草地と奥の苔土手を斜めに見る。
- どんぐりの こみち: やや高い位置から斜面を見下ろし、道を左下から右上へ曲げる。
- せせらぎの みずべ: 水面に近い低いカメラ。砂礫州と対岸を横に見渡す。

外装は背景ごとに草・落ち葉土・湿った岸辺を描き分ける。遠景用と近景用で形を変えるが、独立した洞窟、盛り上がった苔山、切り株の家、石の輪にはしない。外装は低い横長のくぼみとし、斜面の向きは背景側の接地面で吸収する。背景への共通`filter`は使わず、キャラ窓と操作面は回転させない。

## 7. 動物の出し方

各場所は6種類の通常動物を持つ。ランダムの偏りで「場所が変わったのに同じ子しか出ない」と感じさせないため、単純な完全ランダムではなくシャッフル袋を使う。

1. その場所の通常動物6種類を一度ずつ袋へ入れる。
2. 通常出現のたびに1種類取り出す。
3. 6種類が一度ずつ出たら袋を作り直す。
4. 実出現7体目は現状どおり、ひかりモモンガにする。
5. 袋の先頭が現在出現中の動物なら、捨てずに袋の後ろへ回して延期する。
6. 同時に同じ動物を2体出さず、直前の動物も可能なら避ける。

これにより、初めての場所では早い段階で顔ぶれの違いが必ず伝わる。awake／sleepingは同じ動物の2状態で、どちらが出るかは現行のねむり比率に従う。

## 8. 散歩道と結果画面

### 初回の一本道

- 未到達地点を鍵で隠さない。淡い景色の輪郭や葉っぱの点で「この先がある」と見せる。
- 結果カードに別の大きなカードを重ねず、点数・最大コンボの下へ細い散歩道を1本だけ足す。
- 568×320の結果画面では、散歩道を高さ約40pxの3〜5点と`つぎは みずべ`の1行に絞り、全景地図を表示しない。
- 足あと演出は1〜1.5秒以内。タップで短縮できる。
- 足あとが動いている間も主ボタンは押せる。待たせず、押されたら演出を完了状態へ飛ばして次を読む。
- `prefers-reduced-motion`では足あとを即時に次のマスへ置く。
- 各地点の完了時に保存するので、5回連続で遊ぶ必要はない。

568×320の結果カードは、左をポノ、右を次の順で積む。

1. bannerは通常`ナイス ハイタッチ！`、0点時だけ`さいごまで あそべた！`へ置換する。説明行を追加しない。
2. 点数を大きく1行。
3. `さいだい 8コンボ　きろく 12コンボ`を1行へまとめる。
4. 高さ約40pxの散歩道と`つぎは みずべ`。
5. 44px以上の`さんぽを つづける`を1ボタンだけ置く。

開始画面も説明を足さず、現行の2行説明を`1/5　こもれびの ひろば`と散歩の短い一言へ置き換える。

### 5面完走後

結果カードで`つきの はなが さいた！`を表示する。花はプレイ中に常設せず、開花は短い結果演出だけにする。高コンボ時は光の粒を少し増やしてよいが、花の大きさ、物語の完了、次の機能解放は点数に依存させない。

自由選択を追加する段階では、初回完走後に次の2つを解放する。

- `ばしょを えらぶ`: 大きな背景サムネイルから好きな場所を選ぶ。
- `おまかせ さんぽ`: 5場所を順番に巡る。

自由選択画面のサムネイルは背景画像の切り抜きを再利用し、専用画像を増やさない。周回報酬を入れる場合も、消えない葉っぱの足あと1個程度に留める。

連続ログイン、期限付き報酬、低確率キャラ、取り逃し記録、進行減少は入れない。再プレイを急かさず、次の景色が確実に見える構造にする。

### 実装上の変更点

- ゲーム固有の`js/locations.js`を新設し、`locations.js → logic.js → game.js`の順で読む。
- `L.HOLE_COUNT = 6`固定を、選択場所の`slots.length`へ置き換える。
- `pickHole()`は穴数を引数で受け取る純粋関数にし、4・5・6か所を同じテストで通す。
- `#board`の固定gridではなく、正規化した`x / groundY / scale`を持つ絶対配置にする。
- 外装ごとのalpha実体下端を`groundAnchorY`として持ち、正方形画像の中心を接地座標に使わない。
- `.hh-hole`の操作面自体はtransformせず、外装とキャラ窓だけを`scale`で遠近表示する。
- 動物画像はグローバル登録し、場所定義はIDだけを持つ。
- 場所切替時は旧スロットと旧画像参照を破棄してから新しいDOMを作り、前の場所の動物が残らないようにする。
- CSSの`#stage`と`#start-screen`に重複している固定背景をやめ、単一の`--location-bg`または背景画像レイヤーを正本にする。
- セーブ正規化、現在地決定、必須画像decode、背景と盤面の同時交換が終わるまでready表示を出さない。現行の無条件2回requestAnimationFrameによるFOUC解除は、このready条件へ統合する。
- 遊んでいる30秒の間に次の場所だけを先読みし、全場所のawake／sleepingを起動時に一括取得しない。
- 場所を高速に切り替えた時、古いdecode完了や遅延retractが新しい場所を上書きしない`boardEpoch`を持つ。
- 必須画像のerrorまたは4秒のtimeoutでは永久待機にせず、`よみこみが うまくいかなかったよ`と`もういちど`を表示する。再読込または既存ホームへ戻れる終端を必ず持つ。

現行テストにも「穴は常に6個」「上3／下3を`slice()`で分ける」「CSSの先頭3個が遠景」「素材数は固定」という前提がある。Phase 0で本体だけを可変化せず、E2Eと静的回帰も場所定義から期待値を作る形へ同時に直す。

## 10. セーブ

新規キー候補は`pono_hyokkori_walk_v1`。

```js
{
  version: 2,
  routeId: 'mori-5-v1',
  routeCompletedRuns: 0,
  completedLocationIds: [],
  mode: 'route',
  selectedLocationId: null,
  lastCompletedRunId: null,
  locationRecords: {
    komorebi_clearing: { plays: 0, bestScore: 0, bestCombo: 0 }
  }
}
```

- 既存のハイスコアID `hyokkori-hightouch-v2` と最大コンボキーは変更せず、過去記録を維持する。
- 散歩進行と場所別記録だけを新規キーへ保存する。
- 一本道の現在地は`routeCompletedRuns % route.length`、周回数は`Math.floor(routeCompletedRuns / route.length)`から導出し、複数の数値を別々に更新しない。
- run開始時に`runId`、`runLocationId`、`runMode`を固定し、途中で場所選択や保存値が変わってもそのrunの対象を変えない。
- 30秒終了時に`finishGame()`の先頭で一度だけ進行を確定する。保存直前に現値を再読込し、`lastCompletedRunId`が同じなら何もしない。結果ボタン連打では再保存しない。
- `completedLocationIds`は開始画面で見ただけでは増やさず、その場所を30秒完走した時だけ追加する。
- キーがない既存ユーザーは`こもれびの ひろば`から開始する。
- JSON破損、未知の場所ID、範囲外indexは安全に最初の場所へ正規化する。
- localStorageが使えない時も、その起動中はメモリ上で次の場所へ進める。
- 第2弾では`routeId`を`mori-5-v1`へ更新する移行処理を1回だけ通す。旧`routeCompletedRuns`が3以上なら新値を3にして、次を4番目の`きのこの おか`にする。場所別plays／bestと完了済み3場所は保持し、4・5番を完走するまで5場所の自由選択は解放しない。
- 自由選択の完走は場所別記録へ加えるが、一本道の`routeCompletedRuns`は進めない。

4穴と6穴では探索負荷が同一にはならないため、場所別`bestScore / bestCombo`を新しい記録の正本にする。第1弾では画面に増やさず保存だけ行い、Phase 2の自由選択画面で各場所の記録を見せる。既存の全体ハイスコアは過去互換の参考記録として残し、散歩進行や場所解放の条件には使わない。

## 13. 完了条件

### ゲーム

- 30秒、得点、コンボ、7体ごとの30点ボーナス、最大同時2体が全場所で同じ。
- 4・5・6か所すべてで、空き候補、直前成功地点の除外、2体同時出現が正しく動く。
- 各場所の最初の通常6出現で、ロスター6種類が一度ずつ選ばれる。
- awakeへハイタッチ、sleepingは待つという主操作・予備判断が変わらない。
- 場所の違いがスピードアップや当たり判定縮小になっていない。

### 散歩進行

- 0点、低コンボ、ボーナス取り逃しでも完走すれば1マス進む。
- 途中退出では進まず、再開時は同じ場所。
- 終了処理やボタン連打で二重進行しない。
- 壊れたセーブ、存在しない場所ID、場所追加後の旧セーブで停止しない。
- Phase 1を一周済みのセーブは、Phase 2初回に4番目の場所へ進み、4・5番完走前に5場所の自由選択を解放しない。
- 自由選択の完走は場所別記録だけを更新し、一本道の次地点を変えない。
- 既存の全体ハイスコアと最大コンボを失わない。
- 5面完走時は得点に関係なく`つきの はな`が咲き、低得点で物語が止まらない。
- 初めて遊ぶ子が開始と結果の短い文言だけで「種を月の花へ届ける旅」と理解できる。
- 場所別ベストが4・5・6穴それぞれに独立して保存される。

### レイアウト・画像

- 568×320、844×390、1024×768、1366×768で、全かくれ場所がHUDと中央コンボ安全帯を避ける。
- 390×844では横向き案内が全面を覆う。
- 全場所でタップ領域44px以上、目標56px以上。
- awake／sleeping全動物とひかりモモンガについて、停止時の下端crop、出現途中の下漏れ、外装との位置ずれがない。
- 夕方・月夜で、黒いからす、ふくろう、ねむりFX、暗い外装の輪郭が背景へ沈まない。
- 背景と全予定slotへ外装を重ねた合成プレビューで、接地ずれとcover cropがない。
- 背景、外装、動物の遅いdecode中に開始できず、読込後は無反応にならない。
- 背景はdecode完了後に盤面と同時交換され、緑の代替色や前の場所が一瞬見えない。
- 必須画像のerror／timeoutで永久停止せず、かなの再読込UIとホーム導線が出る。
- 場所切替後に前の背景、外装、動物、タイマーが残らない。
- 現在地と次の場所より先の動物画像を過剰に先読みしない。
- 568×320の開始・結果で文字、散歩道、44px主ボタンがoverflowせず、`さんぽを つづける`からタイトルを挟まず次のcountdownへ進む。
- 新規assetのゲーム内cache-bustとService Worker更新を揃え、更新直後と一度読込後のofflineで旧背景・新動物が混在しない。
- reduced-motion、touch、mouse、キーボード、回転・復帰を回帰する。
