# おべんとうチュートリアル 現行ナレーション原稿

> **ステータス: 配信用37本を生成・配置・照合済み。現行シナリオでは36本を使用。**
> 2026-07-13 時点の `bento/index.html` の現行シナリオと実音源に対応する正本です。
> `BENTO_TUTORIAL_MOCK_VOICE` は `false` で、`tut2_*.mp3` と3色ガイドを同じTTS条件で配信します。

## 1. 読み上げ方

- 女性の第三者ナレーターとして、やさしく、少しゆっくり読む。
- ポノや動物の声にはせず、「わたし」「ぼく」などのキャラクター一人称は使わない。
- 子どもが聞く文は、ひらがな・カタカナで読む。「タップ」は使わず、すべて「タッチ」にする。
- 一つの音声では、基本的に一つの操作だけを案内する。操作が終わるまで次の音声を重ねない。
- 収録時は Gemini Leda を 1.15 倍速で使い、生成後に faster-whisper で原稿と照合する。

## 2. 導入の再生順

「じぶんでつくる」では、次の順番を固定します。

1. `tut2_01_greet`: ポノの おべんとうやさんで、おべんとうを つくってみよう。「はじめる」を おしてね。
2. `sk_intro_01`〜`sk_intro_05`: あか・きいろ・みどりの3色ガイド
3. `tut2_02_box`: きょうは、この おべんとうばこを つかうよ。「これで オッケー」を おしてね。

全37本は、同じ `gemini-3.1-flash-tts-preview`、女性の第三者ナレーター `Leda`、1.15倍速、同じ読み上げ指示で生成します。基本の36本は、長い一括生成で後半の声色が変わるのを防ぐため、工程ごとの短い連続ブロックから採用します。`sk_intro_02` だけは「あかは」の発音が自然だった直前の同条件版へ戻しています。既存の3色ガイドは原稿と voice id を維持します。

おねがいモードでは、3色ガイドは既存の開店シーン側の導線で開始カードより前に表示される場合があります。同じ3色音源を使い、同じ案内を二重には再生しません。

### メイン2個目の案内

1個目によって2個目の指定が変わるため、ここは録音せず画面表示だけにします。

- 1個目がハンバーグ以外: `もう ひとつ ハンバーグを えらぼう！`
- 1個目がハンバーグ: `もう ひとつ おかずを えらぼう！`（メインなら自由に選べる）

## 6. 音声にしない動的な案内

次の文は状態や残り回数で変わるため、画面の吹き出しとトーストだけにします。

- `あと 3こ` など、小さいおかずの残り数
- 仕切りの残り数
- メイン2個目の品名
- `それは ちがうよ` から始まる誤操作案内
- `できたね`、`ぜんぶ そろったね` などの途中確認
- レタスを別の場所へ置こうとしたときの、対象名入り再案内

`tut2-leaf-target` はすぐに `tut2-leaf-pick` へ進む内部ステップなので、専用音声は作りません。

## 7. 音声実装記録

- 2026-07-12 の初回収録は34本の約167秒マスターと追加cueで構成していたが、音響監査で目・口の追加takeとの境目、および「はっぱ」工程から終盤にかけて声色と音量のドリフトを確認した。直前の「お気に入り」修正では、それより前の音源ファイル自体は変更していない。
- 同日、全37原稿を工程ごとの4つの短い連続ブロックへ組み直し、各ブロックの先頭・末尾に同じ照合用アンカーを置いて複数候補を生成した。B3はAPIが正順の入力を受け付けなかったため段落だけ逆順で生成し、分割後にシナリオの正順へ戻した。ブロック内アンカーの話者類似度、導入ブロックとの話者類似度、基本周波数・スペクトル、全文の読みを候補選定に使い、`B1_c1` / `B2_c3` / `B3_c2` / `B4_c2` を採用した。追加語や別の締め文が入った候補は採用しなかった。
- 採用したブロックからアンカーを除いて37本へ分割し、全ファイルへ同じ1.15倍速とラウドネス補正を適用した。各音源は24 kHz、mono、96 kbps MP3。実測ラウドネスは -18.88〜-18.73 LUFSで、37本間の幅は0.15 LU。
- faster-whisper `large-v3-turbo` で、初回採用ブロック全体、分割後37本、工程順に連結した確認用音源を照合し、順番、原稿の欠落、各音源の先頭・末尾、余分な案内やラベルがないことを確認した。初回版の分割単体は36/37本が正規化読みで完全一致し、当時の`tut2_06_noriedit`だけは連結ASRとword timestampを併用して原稿一致を確認した。現行の同cueは後述の自由編集原稿へ再収録済み。
- ユーザーの聴感確認を受け、`sk_intro_02` だけbatch1263で配信していた同モデル・同じ`Leda`・同じ1.15倍速の音源へ復元した。SHA-256は `cf7d133330e34fc09126a7397d3fe29337af0ca89c26bc3efeab2d2e08fadd13`、5.208秒、24 kHz、mono、96 kbps、-19.1 LUFS / -3.9 dBTP。faster-whisperで原稿全文を再照合し、前後の現行音声との話者類似度も確認した。現行37本内の音量差は約0.4 LUに収まり、さらに古い版は「おにくや おさかなや」と原稿が異なるため採用していない。
- `tut2_06a_rotate` は旧「縮小→時計回り」工程との互換用にファイルだけ保管し、現行シナリオからは切り離す。旧 `tut2_06b_editundo_last` も使用しない。`tut2_06b_editundo` の音声終了後は、操作回数にかかわらず自由編集開始時の保存状態へ一度で復元する。
- 2026-07-13、`tut2_06b_editundo` と `tut2_10b_freeokazu` を現行シナリオに合わせて再収録し、`tut2-pick-move` 用の `tut2_11e_pick_move` を追加した。使用中音声は旧取り消し2本目との入れ替えで全37本を維持する。
- 上記3本は `gemini-3.1-flash-tts-preview` / `Leda` の共通アンカー付き3候補から、既存承認音声とのMFCC・声高・スペクトル・RMS比較で候補3を採用した。全て1.15倍速、24 kHz、mono、96 kbps、-18.84〜-18.80 LUFS。faster-whisper `large-v3-turbo` の個別照合は3本とも正規化読みで完全一致。SHA-256は `tut2_06b_editundo` = `8d826602a38022505106ab4d07e4474a6813f118414a8b5daea7264fa35e7acd`、`tut2_10b_freeokazu` = `62b466dc468df156f9308de421eade1659ffb4bb3a9429f3a49fe319ffab09d6`、`tut2_11e_pick_move` = `ff8d1863843a0ecb2bf172cff9ef7993335a24f9ddbf012c047fbe8500092c91`。
- 2026-07-13、`tut2_06_noriedit` を4つの編集ボタンを自由に反復する現行原稿へ再収録した。`gemini-3.1-flash-tts-preview` / `Leda`、1.15倍速、24 kHz、mono、96 kbps、11.890秒、-18.80 LUFS / -5.32 dBTP。faster-whisper `large-v3-turbo` は正規化読みで完全一致。SHA-256は `f2e8d94c3be65b891eb013256a64af338806246fd12d9eaf44192bbca2ac565b`。前後3本とのMFCC類似度は0.9973〜0.9992で、既存話者との声質連続性も確認した。
- `tut2-nori-edit`、`tut2-nori-ok`、`tut2-free-okazu`、`tut2-leaf-tab`、`tut2-divider-place`、`tut2-pick-tab`、`tut2-pick-place`、`tut2-pick-move` を voice map へ接続する。
- 原則として音声の再生終了だけでは操作工程を進めず、対応するタッチや配置の完了を待つ。例外は `tut2-nori-undo` で、説明終了後に見本側の保存状態復元を始める。音声中に他の操作が完了した場合は、残り再生時間に安全余白を加えてから次工程へ進む。
- `BENTO_TUTORIAL_MOCK_VOICE` を解除し、配信用37本（現行使用36本）を `v1278-color-nori-free` の cache-buster 付きで実音源として配信する。
