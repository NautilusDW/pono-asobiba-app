## 2. リライト方針の要約 (スタイルブリーフ抜粋)

編集長 (fable) が3方針案のコンペ・採点を経て確定したブリーフ。全7対象共通。

- **二層レジスタ**
  - 層A = モーダル内シーンキャプション (5ゲーム×3枚+特典2枠×2枚=計17枚、現行実装分): 読者は子ども (親が読み聞かせる想定)。純ひらがな・カタカナ+分かち書き、漢字混入禁止。女性ナレーター第三者視点の話しかけ型、文末は「〜よ / 〜ね / 〜しよう / 〜できるよ」。キャラの一人称セリフは禁止 (CLAUDE.md Base Rule 6)。モード名・お店名は「」で括る。
  - 層B = growth文 (カード側 `.pc-growth` + モーダル側 `GAME_MODAL_DATA.*.growth`、常に同一文言で両方同時更新): 読者は保護者。敬体 (です/ます)・漢字使用可。「〜が育ちます」結びを全7スロット維持。`<b>` タグの力ワードは**2個まで** (現行 oto の4連は反面教師として明記)。
- **三幕アーク公式 (5ゲーム3枚構成)**: 3枚を通しで音読すると1つの短い物語になること。S1=であい・さそい (世界への入口+最初のアクション)、S2=あいてのいる核 (相手キャラ+子どもの判断/工夫+喜びの反応)、S3=じゆう・あんしん・ごほうび (自由創作+安心設計+継続フック)。「失敗しても だいじょうぶ」系フレーズを3枚のどこか1箇所に必ず入れる。特典2枠は圧縮アーク (2枚構成: 儀式→サプライズ / 行動ループ+保証の仕組み)。
- **合格テスト3点 (全キャプション必須)**
  1. 転用不能テスト: ゲーム名を隠して読んでどのゲームか当てられなければリテイク
  2. 実装トレーステスト: 文中の全事実がコード実証済み+無料tierでローンチ済み (comingSoon/app限定機能の訴求は禁止。bento のトントンキッチン下ごしらえは特に訴求禁止と明記)
  3. 読み聞かせテスト: 3枚通しで1つの物語として音読できること
- **文字数ルール** (全角換算、375pxモバイルで4行以内が絶対条件)
  - 層A (caption): 35〜70字、88字が絶対上限
  - 層B (growth): `<b>` タグ除去後40〜65字 (カード側 `.pc-growth` の狭い幅が基準)

## 4. 各コピーの根拠にした実装ファクト

- **bento**: 「おみせモード」はお客さん動物6種の注文制 (NPC_REQUESTERS)、三色バランスは星1〜3+どんぐり2〜5個の5段階自動判定。「じゆうモード」ののり細工・かざりパーツは72種 (うち電車・虹などの「のり形状」27種) — growth の「のり細工で自分らしく」の根拠。
- **maze**: おじゃまムシは9種類 (`CREATURE_KIND_DATA`)、それぞれじゃんけん/○×クイズ/リズムまねシモン等のミニゲームに対応。行き止まりでは一つ前の道に戻ってやり直せる設計 (S3「もどって、 やりなおせるよ」の根拠)。
- **oto**: 演奏ボタンは8個 (ド〜高いド)。リズムモードのタイミング判定は5段階、3体のライバルと対戦、コンボ8連続でライバルが「パニック」状態になる (S2「あいての かおが かわるよ」の根拠)。
- **puzzle**: パートナー (なかま) は8種、うちキツネはヒントで正解シルエットを1.5秒表示、こじかは近づくとピースが光る (グロー+磁石引き寄せ) — なかまごとに「助け方」の仕組みそのものが異なる (見た目替えではない)。
- **quizland**: 出題テーマは8種×3難易度=全180問、1プレイ5問構成。はかせのふきだしタップで段階的ヒント、まちがえても選び直せるノーペナルティ設計。正解時に57問でシルエット→カラー写真のリビール演出。
- **feature-gacha**: レバーは3回転で1回のガチャ (`DAILY_GACHA_TURNS=3`)、1日最大2回 (1回目無条件+2回目はデイリークエストクリア条件)。カプセル開封時に `dailyGachaOwnershipSummary` ではじめて/重複を明示。どんぐりショップは未所持シール優先の重み付き抽選 (`guaranteedPool`) + 「とりおき」予約1点まで。
- **feature-sticker-book**: シールちょうは Three.js 製の本物の物理演算3Dページめくり本。お絵かき機能はペン15色・スタンプ8種。シールミュージアムは7つのテーマ別展示室+ロック中の2階1つ (「じゅんびちゅう」表記の根拠)。
