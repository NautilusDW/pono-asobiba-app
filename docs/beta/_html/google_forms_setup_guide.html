ポノのあそびば βテスト Google Forms 設定手順書

想定所要時間: 約10分 × 2本 = 20分 並走するコピペシート: google_forms_template.md 困ったら末尾 FAQ / トラブルシュート へ。

0. 用語ミニ辞典 (最初に 30秒)

用語 意味
Google Forms Google のアンケート作成ツール。 ブラウザだけで使える、 無料。
質問形式 (回答形式) ラジオボタン / チェックボックス / 記述式 / 段落 / 均等目盛 など、 回答の入力方法。
ラジオボタン 1つだけ選べる丸印。 「単一選択」。
チェックボックス 複数選べる四角印。 「複数選択可」。
記述式 (短文回答) 1行のテキスト入力。
段落 (長文回答) 複数行のテキスト入力。
均等目盛 1-5 や 0-10 のような 段階評価のバー。
セクション 質問のグループ分け。 ページ区切りにもなる。
事前入力リンク URL に値を埋め込んでおいて、 回答者が開いた時に最初から入力された状態にする仕組み。 fid 自動付与に使う。
entry ID 各質問の内部 ID (例: entry.123456789)。 事前入力リンクに必要。
応答 (Responses) 集まった回答データ。 スプレッドシートに自動連携できる。

1. 中間アンケートを作る (10分)

Step 1-1. 新規フォームを作成 (1分)

  1. ブラウザで を開く
  2. 左上の 「空白のフォーム」 (+ アイコン) をクリック
  3. 画面上部の 「無題のフォーム」 をクリック → タイトルを以下に変更: ``` ポノのあそびば おためし版 中間アンケート ```
  4. タイトル直下の 「フォームの説明」 をクリック → コピペシートの「説明文」 をペースト

Step 1-2. 匿名化設定 (1分)

  1. 画面上部 「設定 (Settings)」 タブをクリック
  2. 「回答」 セクションを展開:
    • 「メールアドレスを収集する」 → OFF (回収しない)
    • 「ログインを必須にする」 → OFF (Google アカウントを要求しない)
    • 「回答を 1 回に制限する」 → OFF (※ON にすると Google ログイン必須になるため、 βでは OFF 推奨。 fid で重複検出する設計)
    • 「プレゼンテーション」 セクションを展開:
    • 「進行状況バーを表示」 → ON
    • 「確認メッセージ」 にコピペシート末尾の確認メッセージをペースト

Step 1-3. 質問を順番に追加 (5-6分)

  1. 上部 「質問」 タブに戻る
  2. デフォルトで 1つ目の質問枠が表示されているので、 コピペシートの Q0 (fid) から順に作成
  3. 各質問の作成手順:
    1. 質問テキスト欄にコピペシートの「質問タイトル」 を貼る
    2. 右上の 質問形式プルダウンをクリックして、 コピペシートの「回答形式」 を選ぶ
    3. 選択肢タイプ (ラジオ/チェックボックス) なら 「選択肢を追加」 を繰り返してコピペシートの選択肢を 1行ずつ貼る
    4. 右下 「必須」 トグルをコピペシートに従って ON/OFF
    5. 質問枠の右下 「︙」 (3点メニュー) → 「説明 (Description)」 を ON にして、 ヘルプテキストを貼る (必要な場合のみ)
    6. 右側ツールバーの 「+」 (質問を追加) をクリックして次の質問へ
    7. Q5 (最終問) まで完成したら、 画面右上 「目のアイコン (プレビュー)」 で全体確認

Step 1-4. fid の entry ID を取得 (★ 重要・2分)

fid を URL から自動付与するために必要な作業。 1回だけで OK。

  1. 画面右上 「︙」 (3点メニュー) をクリック
  2. 「フォームに事前入力する」 をクリック (英: Get pre-filled link)
  3. プレビュー画面が開く → Q0 の fid 欄に仮の値 SAMPLE を入力 (他の必須欄も適当に埋める)
  4. 下部の 「リンクを取得」 ボタンをクリック
  5. 上部に出てくる URL を 「リンクをコピー」 でコピー
  6. メモ帳などに貼り付けて、 entry.数字=SAMPLE の部分を探す
    • 例: https://docs.google.com/forms/d/e/xxx/viewform?entry.1234567890=SAMPLE&entry.9876543210=...
    • この entry.1234567890 が fid 用の entry ID
    • entry.1234567890 を控える (この後 fid 発行スクリプト/配布物 URL に使う)

Step 1-5. 公開 URL を取得 (1分)

  1. 画面右上 「送信」 ボタンをクリック
  2. 上部タブで 「リンク」 アイコン (鎖マーク) を選択
  3. 「URL を短縮」 にチェックを入れると https://forms.gle/xxxxx の短い URL が得られる (任意)
  4. 「コピー」 で URL を取得 → 控える

Step 1-6. スプレッドシート連携 (1分)

  1. 画面上部 「回答 (Responses)」 タブをクリック
  2. 「スプレッドシートにリンク」 (緑のアイコン) をクリック
  3. 「新しいスプレッドシートを作成」 を選択 → デフォルト名でOK → 「作成」
  4. 新しいタブでスプレッドシートが開けば成功 → 以後、 回答が自動で行追加される

画像ファイルの準備 (★ 着手前に必ず実施・2分)

  1. 入手元: GitHub リポジトリ assets/ui/title/ フォルダ

    • 均等目盛 (Q1, Q5, Q8, Q9, Q11) の設定手順:
    • 質問形式プルダウンで 「均等目盛」 を選択
    • 数値範囲を指定 (Q11 のみ 0-10、 他は 1-5)

4. FAQ / トラブルシュート

Q. 「フォームに事前入力する」 が見つかりません

A. 画面右上の 「︙」 (縦3点メニュー) の中にあります。 質問編集中ではなく フォーム全体の編集画面 (URL が /edit で終わる画面) で開いてください。 セクション内の質問の「︙」 とは別物です。

Q. fid 欄を回答者に編集されたくない

A. 残念ながら Google Forms には「読み取り専用フィールド」 機能がありません。 ヘルプテキスト (このコードは自動で入力されます。 そのまま次に進んでください。 (空欄でも回答できます)) で抑止し、 集計時に短縮 URL とのクロスチェックで弾く運用にします。 万一改ざんがあっても fid 長 6文字 base32 + 配布リスト突き合わせで検出可能です。

Q. 画像をアップロードしたのに 「壊れた画像」 マークが出る

A. 画像形式が webp の場合、 まれに Forms が表示できないことがあります。 PNG または JPEG に変換してから再アップロードしてください (このプロジェクトは assets/ui/title/ 下に PNG 版があれば PNG を優先)。

Q. 「回答を 1回に制限」 を ON にできない

A. Google アカウントへのログインが必須になる設定のため、 βテストでは OFF が正解です。 重複回答の検出は fid (匿名 ID) で行います。

Q. スプレッドシート連携を忘れたまま回答が来てしまった

A. 後から連携しても 過去の回答も自動で取り込まれるので大丈夫です。 「回答」 タブ → 「スプレッドシートにリンク」 を後から実行してください。

Q. 質問の順番を入れ替えたい

A. 質問枠を マウスでドラッグ (枠の上端の点々が並んだ部分をつかむ) すれば移動できます。 ただし fid 質問は 必ず先頭 に置いてください (entry ID は変わりませんが、 回答者が混乱しないため)。

5. 完成後の運用フロー (参考)

  1. Day 0 (配布開始): チラシ/同意書/アプリ起動時に ?fid=XXXXXX 付き URL を配布
  2. Day 7 (中間アンケート配信): アプリ内バナー + メール で中間アンケート URL (fid prefill 付き) を案内
  3. Day 14 (最終アンケート配信): 同様に最終アンケート URL を案内
