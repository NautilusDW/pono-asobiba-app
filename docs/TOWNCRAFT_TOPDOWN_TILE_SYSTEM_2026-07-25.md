# まちづくり（TownCraft）2Dトップダウン仕様

## ねらい

昔の2Dレトロゲームのような真上寄りの画面で、土・草・木を中心に、川、道、丘、段差、建物、小物を組み合わせて町を作る。部屋のアイソメトリック表現とは切り離し、町フィールドは通常の正方形タイルで構成する。

## 「わたしのおうち」

- 固定の「ポノの家」ではなく、プレイヤー本人の家。
- 表札はプロフィール名から `〇〇の おうち` と表示する。空欄は `わたしの おうち`。
- 家を選び直しても、入口、部屋データ、図鑑への導線は同じ家IDを使い続ける。
- 家の中は別シーンの「わたしの おへや」へ接続する。家具配置と図鑑閲覧は接続口を実装済みで、既存のおへや／図鑑シーンを統合するときに遷移先を結ぶ。

## タイルと配置レイヤー

マップは18×12セルのデモ構成。地形データと見た目を分離する。

- 地面: 草、土、道、水、高さ
- 接続タイル: 道と川は上下左右の隣接マスクで直線・角・T字・十字・終端を選択
- 高さ: セルごとの段差。丘の縁には崖タイルを重ねる
- 建物: 道に隣接した建物セル
- 道沿い: 街灯、ベンチ、掲示板、おすそわけ台など。建物スロットと別
- 植物: 木、花、低木。道や水以外
- 境界: 柵、門。建物とは別
- 水辺: 橋、桟橋、水辺装飾

これにより、家の場所へ街灯が置かれるような混在を防ぐ。

## 畑から町が育つ理由

野菜を直接「建物通貨」にはしない。畑は住民との関係を生む入口にする。

1. 野菜を育てる
2. 掲示板の「まちの おねがい」へ届ける／おすそわけ台へ置く
3. 住民から木材・石材・布などの「ざいりょう」と、新しい設計の「ひらめき」を受け取る
4. 材料で設備を作り、ひらめきで新しい家や町テーマを開放する
5. 店や公園が増えるほど、新しい依頼と種が町へ戻る

実装済みの依頼はトマト、たまねぎ、じゃがいもの3種類。報酬の二重受取を防ぐ保存ロジックも含む。

## テーマ交換

セルの地形・配置・保存データは共通で、画像だけを `Resources/TownCraft/Themes/{Theme}/` から引く。現在のCountryside素材がない場合も共通素材へフォールバックする。

- Countryside: 土、草、木、素朴な建物＋一部の近代設備
- Modern: 舗装、街路樹、現代的な標識・街灯・住宅
- Future: 発光路面、未来の水路、ホログラム設備

同じ町のレイアウトを保ったままテーマを切り替えられる設計で、近代都市化のためにシステムを作り直す必要はない。

## Unity上の操作

- `じめん / みち / かわ / たかく / ひくく / けす`
- `しょくぶつ / みちのそば / さく / おうち`
- 家の種類送り、テーマ切替、保存
- `おへや` と `ずかん` は遷移接続口

子ども向け画面文言は漢字を使わず、かな表記にしている。

## 素材規格

- タイル: 256×256 PNG、正方形、継ぎ目が合うこと
- 小物: 256×256 PNG、透明背景、足元中央を基準
- 家: 384×384 PNG、透明背景、正面入口と空の表札
- Pixels Per Unit: 256
- Filter Mode: Bilinear
- Compression: None

GPT Image 2の生成シートをマスターとして残し、セル分割したUnity素材も同梱する。将来は同じファイル名でModern/Futureパックを追加できる。
