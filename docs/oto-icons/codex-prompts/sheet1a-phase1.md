# Codex 発注プロンプト: 音タッチ Sheet 1a Phase1 (楽器ボタン デザイン言語提案)

あなたは子供向けゲーム UI 用ボタンアセットシートのデザイナーです。ドレミ 8 音の楽器ボタン 8 個を 1 枚に並べたアセットシートを、**デザイン言語 (質感・形・ハイライト・シャドウ・グロー・線処理・全体ムード) を案ごとに大きく変えた 4〜5 案**として提案してください。前回フィードバック: 「中央アイコンだけ違って質感が全部同じ」だと提案として弱い。今回は**ボタンのデザイン言語そのものを別物にする**こと。

## ゴール
8 個揃いの楽器ボタンシート (1024×512 / 4×2 / 透過 PNG) を **4〜5 案、各案 1 枚ずつ独立した PNG** として書き出す。各案は別物の設計言語で、ユーザーが 1 案を選んで Phase2 (最終発注) に進む。

## 世界観・トーン (全案共通)
- ポップ、カラフル、子供向けゲーム UI、ただし**洗練を保つ** (幼稚すぎないこと)
- 明るく彩度高め
- ポノ (クマキャラ) は描かない (音タッチ画面にはキャラ世界観を持ち込まない)
- **絵本水彩・木製・布地・ステッチ・フェルト・紙質感・ヴィンテージ調・くすみ色・アースカラーは禁止**

## シート寸法と座標系 (全案共通)
- サイズ: **1024 × 512 px** PNG、背景は完全透過 (#00000000)
- グリッド: **4 列 × 2 行 = 8 セル**、各セル 256 × 256 px、内側パディング 18 px
- safe area: **220 × 220 px** (被写体はこの中に収める)
- セル左上座標公式: `x = (col-1)*256 + 18`, `y = (row-1)*256 + 18`
- 描画アンカー: 各セル中心揃え (`cellCenter = (x + 128, y + 128)`)。被写体は中心から左右上下対称
- 実描画: ボタン外径 **200〜220 px** をセル中央に配置 (外形が円でない案でも、外接矩形はこのサイズ内)
- 8 セルすべて使用 (空きセルなし)

## デザイン言語を案ごとに変える (重要)
**質感・形・ハイライト・シャドウ・グロー・線処理・全体ムードまで含めて、案ごとに別物の設計言語で 4〜5 案**を出してください。同じテーマで角度違いの微調整は禁止。

### 設計言語の例示 (誘導しすぎない範囲、Codex の判断で 4〜5 種類を選ぶ)
- グロッシージェル / キャンディ (光沢ハイライト・透明感・柔らかい影)
- ネオン・サイバー (発光リング・グロー主体・電子的シンボル、黒背景前提でなくとも透過上で成立するネオンライン)
- フラット + 太いトゥーンアウトライン (黒〜濃色の太線・ベタ塗り・最小限の影)
- 3D プラスチック / iOS リッチアイコン風 (奥行きのあるベベル・複数ハイライト・接地影)
- 半透明グラスモーフィズム (薄いガラス層・縁の屈折・内側のぼかし)
- ペーパーカット風 (層の重なり・段差影、ただし"紙質感"そのものではなく**幾何的レイヤー**として扱う)
- クレイ風 (柔らかい立体・マット質感、ただし"粘土感"を強調せずポップに留める)
- レトロアーケード (ピクセル風シンボル・スキャンライン控えめ・蛍光色)
- ピル型 / 六角フレーム / 角丸スクエア など外形違い

上記から **4〜5 種類を Codex の判断で選択**。並べた時にバラエティが効くよう、ジェル系で 3 案被らせるようなことはしない。

### 各案で自由に決めてよい要素
- 外形 (円 / ピル / 角丸スクエア / 六角 / ベベル付き 等)
- ハイライト方向・グロー強度・シャドウ強度・線太さ
- 中央モチーフ (その案の設計言語に合う**抽象モチーフ**を Codex が選ぶ — 音符・波紋・スパーク・幾何形・リング・粒・電子的シンボル等)

## 全案で揃える共通要件 (これだけ守る)
- 1 案内で 8 個並んだとき、**同じ世界の楽器ボタン群というシリーズ感**が出ること (同じ設計言語が 8 個分連続)
- 8 音の色相が上記テーブル通り (虹順、ド=赤 → ド高=ピンク)
- 子供向けポップな印象、洗練を保つトーン

## 絶対禁止
- 既存仮素材の絵文字 (🌸🦋⭐🍀🎈🍎🐟🌈) を踏襲しない — 完全置換
- 花・蝶・果物・動物などの具体物描写 (音色を想起させてしまう)
- 絵本水彩、パステル淡色、木目、布地、ステッチ、紙質感、ヴィンテージ調、くすみ色
- 幼稚に見えるイラスト調 (ゲーム UI として洗練を保つ)
- ポノ (クマキャラ) を各ボタンに描き込む
- 1 枚の PNG に複数案を横並びで入れる (案ごとに独立 PNG)
- 押下／アクティブ状態の別バージョン (Codex は通常状態のみ納品。アクティブ表現は CSS 側)
- **質感が同じで中央モチーフだけ違う案を複数並べる** (今回の最重要 NG)

## Phase1 出力形式
- **4〜5 案を、案ごとに 1 枚の独立 PNG として書き出す**
- 各 PNG は 1024×512 / 4×2 / 8 セル全充填、背景透過 (#00000000)
- ファイル名:
  - `sheet1a_phase1_proposalA.png`
  - `sheet1a_phase1_proposalB.png`
  - `sheet1a_phase1_proposalC.png`
  - `sheet1a_phase1_proposalD.png`
  - `sheet1a_phase1_proposalE.png` (5 案目を出す場合、4 案なら A〜D で停止)

## 質問返し
不明点があれば日本語で質問してください。前提を独自判断で埋めて発注を進めないこと。特に **「どの 4〜5 種類の設計言語を選んだか」**は提案 PNG の納品時に簡潔に列挙すること (例: 「A=グロッシージェル / B=ネオンサイバー / C=トゥーン太線 / D=3D プラスチック」)。
