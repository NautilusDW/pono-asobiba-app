# Codex 発注プロンプト: 音タッチ Sheet 1a Phase1 (楽器ボタン中央モチーフ提案)

あなたは子供向けゲーム UI 用ボタンアセットシートのデザイナーです。以下の条件で、楽器ボタン 8 個を 1 枚に並べたアセットシートを **2〜3 案** 提案してください。

## ゴール
ドレミ 8 音 (ド／レ／ミ／ファ／ソ／ラ／シ／ド高) を押すための円形ボタン 8 個を 1 枚のシートに配置した PNG を、中央モチーフ違いで **2〜3 案、各案 1 枚ずつ独立 PNG** として書き出す。ユーザーが後で 1 案を選び、最終発注 (Phase 2) に進む。

## 世界観・スタイル指針
- ポップでカラフルなゲーム UI、子供向けだが洗練を保つ
- 軽い 3D 感のある円形ボタン (セミフラット、光沢・ハイライト・ソフトグラデ可)
- **絵本水彩・木製・布地・ステッチ・フェルト・紙質感・ヴィンテージ調は禁止**
- くすみ色・アースカラー禁止。彩度高めの明るい色
- ポノ (クマ) のキャラクター本体は描かない (音タッチ画面にはキャラ世界観を持ち込まない)

## シート寸法と座標系
- サイズ: **1024 × 512 px** PNG、背景は完全透過 (#00000000)
- グリッド: **4 列 × 2 行 = 8 セル**、各セル 256 × 256 px、内側パディング 18 px
- safe area: 220 × 220 px (被写体はこの中に収める)
- セル左上座標公式: `x = (col-1)*256 + 18`, `y = (row-1)*256 + 18`
- 描画アンカー: 各セル中心揃え (`cellCenter = (x + 128, y + 128)`)。被写体は中心から **左右上下対称**
- 実描画: 円形ボタン外径 **200〜220 px** をセル中央に配置
- 8 セルすべて使用 (空きセルなし)

## ボタン土台デザイン (共通スタイル基準・全案共通で揃える)
- 円形外径 200〜220 px
- **ハイライト方向は左上 45°** から光が当たる前提 (全ボタン統一)
- **内包ドロップシャドウ**: 不透明度 25〜35% / ぼかし 6〜10 px / オフセット下方 4〜6 px (セル外に影を出さない)
- **グロー**: ベースカラー ±20% で 1〜2 層 (内側 or 外周ソフトグロー)
- 線アクセントを入れる場合: 白系 / 太さ 2〜3 px
- 角丸 (円以外の形を使う場合): 半径 24 px

## 中央モチーフ (案ごとの変動ポイント)
具体物は禁止。**音・光・響きを連想させる抽象モチーフ** (音符、波紋／リップル、輝き／スパーク、雫、リング、星型、幾何形 など) を、**8 個揃いで統一感が出る一系統**でまとめてください。
- 案 A / 案 B / (案 C) でモチーフ系統を変える (例: 「リップル統一」「抽象音符＋輝き」「幾何リング」など、具体案は Codex 判断)
- 8 個並べた時にシリーズとして揃って見え、**色相だけで識別できる構造**
- 案間で「ハイライト方向／グロー彩度／内包シャドウ強度／ボタン土台形状」は **完全に揃え**、変動させるのは中央モチーフのみ

## 絶対禁止
- 既存仮素材の絵文字 (🌸🦋⭐🍀🎈🍎🐟🌈) を踏襲しない — 完全置換
- 花・蝶・果物・動物などの具体物描写 (音色を想起させてしまうため)
- 絵本水彩、パステル淡色、木目、布地、ステッチ、紙質感、ヴィンテージ調
- 子供向けすぎて幼稚に見えるイラスト調 (ゲーム UI として洗練を保つ)
- ポノ (クマキャラ) を各ボタンに描き込む
- 1 枚の PNG に複数案を横並びで入れる (案ごとに独立 PNG)
- 押下／アクティブ状態の別バージョン (Codex は通常状態のみ納品。アクティブ表現は CSS 側で行う)

## Phase1 出力形式
- **2〜3 案を、案ごとに 1 枚の独立 PNG として書き出す**
- 各 PNG は完成版と同じ 1024×512 / 4×2 / 8 セル全充填
- ファイル名:
  - `sheet1a_phase1_proposalA.png`
  - `sheet1a_phase1_proposalB.png`
  - `sheet1a_phase1_proposalC.png` (3 案目を出す場合)
- 背景: 透過 (#00000000)
