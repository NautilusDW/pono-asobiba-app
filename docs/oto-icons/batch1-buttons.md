# 音タッチ ボタン素材シート Batch 1 — Codex 発注仕様書

## 用途宣言（Batch1 / Batch2 の役割分担）
- `assets/images/oto/buttons/` = **タップ対象 UI 素材** (本 Batch1)。ユーザーが押す物理ボタン。土台＋中央モチーフ一体描画。
- `assets/images/oto/icons/` = **装飾 / 粒子素材** (Batch2)。canvas に飛ばすパーティクル / DOM に貼る装飾。タップ非対象。
- prefix 衝突回避: ボタンは `btn_note_*` `btn_set_*` `btn_speed_*` `btn_ui_*`、装飾粒子は `spark_*` `deco_*`。

## ゴール
子供向け音遊びアプリ「音タッチ」のボタン 23 個（楽器 8 + 音色 6 + 速度 3 + UI 制御 6）を、**2 枚のアセットシート（Sheet 1a / Sheet 1b）**に分けて発注する。**ポップでカラフルなゲーム UI**（絵本テイスト・木製・布素材は一切禁止）。アイコンとボタン土台は一体で描き、後で切り出して `assets/images/oto/buttons/` に配置する。

なお Sheet 1a（楽器ボタン）は中央モチーフを Codex に**2〜3 案提案**させる **2-phase 運用**とし、本仕様書はその提案フェーズのスタイル制約のみを定義する（最終確定後に同じ Sheet で本発注）。

## スタイル方針（両シート共通）

### DO
- ポップ、明るい、彩度高め、ゲーム UI 寄り
- 光沢・ハイライト・ソフトグラデ・薄い内側グロー OK
- 円形・ピル型の土台、フラット〜セミフラット
- ボタン本体に**内包**ドロップシャドウ可（セル外への影出しは禁止）。Batch2 のパーティクル素材を後から合成する場合は**影なしで表示**できる前提なので、影は強くしすぎない
- 子供が「押せる」と直感する立体感（軽い 3D ボタン風）

### DON'T
- 絵本水彩、木目、布、ステッチ、刺繍、フェルト、紙、ヴィンテージ調
- くすみ色・アースカラー
- ポノ（クマ）を各ボタンに描き込む（タイトルロゴ専用）
- 既存仮素材の絵文字 (🌸🦋⭐🍀🎈🍎🐟🌈) を踏襲して描かない。**これらは仮素材なので完全に置き換える**
- **アクティブグロー版を同シート内に作らない**（押下/アクティブ状態は CSS で表現する。Codex は通常状態のみ納品）
- 空きセルは**完全透過**で残す（背景・ガイド線・ロゴ等を入れない）

## 共通スタイル基準 (Sheet 1a / 1b / Batch2 すべて共通)

本セクションは Sheet 1a・Sheet 1b に加え、**Batch2 (`batch2-particles.md`) からも参照される正本**。全アセットで以下の数値ガイドラインを統一すること。

- **ハイライト方向**: **左上 45°** から光が当たる前提で描く（全アセット統一。光源が他向きの案は不可）
- **内包ドロップシャドウ**: 不透明度 **25-35%** / ぼかし半径 **6-10px** / オフセット **下方 4-6px**（セル外に影を出さない）
- **グロー彩度**: 高彩度ベースカラー **±20%** の範囲で **1〜2 層** の発光（内側 or 外周ソフトグロー）
- **線アクセント**: 入れる場合は白 or 高明度の単色、太さ **2-3px**（256px 描画時の基準。Sheet 1b の 300×340 セルでは比例して 2.5-3.5px 程度）
- **角丸**: 外形が円でない場合の角丸は半径 **24px 前後**（256px 描画時の基準）

## Sheet 1a — 楽器ボタン（メイン操作 / 一番大きく綺麗に）

### 用途
ドレミ 8 音 (`data-set="doremi"` の 8 ボタン)。アプリのメイン操作面なので最大解像度で発注。

### シート仕様
- サイズ: **1024 × 512 px** PNG, 背景透過 (#00000000)
- グリッド: **4 列 × 2 行 = 8 セル**, 各セル 256 × 256 px, 内側パディング 18 px
- 実描画: **直径 200〜220 px** をセル中央に配置
- 8 セルすべて使用（空きセルなし）

#### bbox 機械抽出公式 / アンカー / safe area (Sheet 1a)
- **セル左上座標公式**: `x = (col-1) * cellW + paddingX`, `y = (row-1) * cellH + paddingY`
- **セル寸法表**:

  | パラメータ | 値 |
  |---|---|
  | cellW | 256 |
  | cellH | 256 |
  | paddingX | 18 |
  | paddingY | 18 |

- **描画アンカー**: 全セル **セル中心揃え** (`cellCenter = (x + cellW/2, y + cellH/2)`)。被写体は中心から左右上下対称に配置
- **safe area**: padding を除いた `cellW - 2*paddingX` × `cellH - 2*paddingY` = **220 × 220 px** を「safe area」と呼び、被写体はこの中に収める

### 中央モチーフ（2-phase 運用 / Codex が複数案提案）
具体的なモチーフは指定しない。**世界観の制約だけを与え**、Codex は「ポップなゲーム UI に合う、8 個揃いで成立する中央アイコン」を **2〜3 バリエーション**を提案する（例：抽象音符系 / 光のスパーク系 / 結晶＋音符ハイブリッド系 など、Codex 自身の判断で）。

制約：
- 既存仮素材の絵文字（🌸🦋⭐🍀🎈🍎🐟🌈）は**完全に置き換える**（仮なので保持しない）
- 花・蝶・果物・動物などの**具体物描写は避ける**（音色を想起させてしまうため）
- 8 個並べたときに**シリーズとして統一感**があり、色相だけで識別できる構造
- ポノ（クマ）を描かない

ユーザーが提案から 1 つ選び、その方針で同じ Sheet 1a を再発注して最終納品とする。

#### 提案フェーズ (Phase 1) の出力形式
- **2〜3 案を 1 案ずつ独立した PNG** で書き出すこと（1 枚に複数案を並べない）
- 各 PNG は完成版と同じ仕様 (1024×512, 4×2 グリッド, 8 セル全て) で、案ごとに楽器 8 個のモチーフバリエーションを揃えて作る
- ファイル名例: `sheet1a_phase1_proposalA.png`, `sheet1a_phase1_proposalB.png`, `sheet1a_phase1_proposalC.png`
- 案間で **ハイライト方向 / グロー彩度 / 内包シャドウ強度** は揃え（共通スタイル基準セクション準拠）、変えるのは **中央モチーフのみ**

## Sheet 1b — その他ボタン（音色 6 + 速度 3 + UI 制御 6 = 15 個）

### 用途
メイン操作以外の補助ボタン群。**Sheet 1a より一回り小さい印象**で描く。

### シート仕様
- サイズ: **1536 × 1024 px** PNG, 背景透過 (#00000000)
- グリッド: **5 列 × 3 行 = 15 セル**（15 個ぴったり、空きセルなし）
- 各セル: 約 300 × 340 px, 内側パディング 20 px
- 実描画: **220〜240 px 直径目安**でセル中央

#### bbox 機械抽出公式 / アンカー / safe area (Sheet 1b)
- **セル左上座標公式**: `x = (col-1) * cellW + paddingX`, `y = (row-1) * cellH + paddingY`
- **セル寸法表**:

  | パラメータ | 値 |
  |---|---|
  | cellW | 約 307 (1536 / 5) |
  | cellH | 約 341 (1024 / 3) |
  | paddingX | 20 |
  | paddingY | 20 |

  > 実装上は `cellW = 300`, `cellH = 340` の **論理セル** として扱い、周囲の余白で帳尻を合わせる運用でも可。bbox 抽出時はシート実寸を 5 / 3 で等分した値を採用する。

- **描画アンカー**: 全セル **セル中心揃え** (`cellCenter = (x + cellW/2, y + cellH/2)`)。被写体は中心から左右上下対称
- **safe area**: padding を除いた `cellW - 2*paddingX` × `cellH - 2*paddingY` ≒ **約 267 × 301 px**。被写体はこの中に収める
