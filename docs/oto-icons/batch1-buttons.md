# 音タッチ ボタン素材シート Batch 1 — Codex 発注仕様書

## 用途宣言（Batch1 / Batch2 の役割分担）
- `assets/images/oto/buttons/` = **タップ対象 UI 素材** (本 Batch1)。ユーザーが押す物理ボタン。土台＋中央モチーフ一体描画。
- `assets/images/oto/icons/` = **装飾 / 粒子素材** (Batch2)。canvas に飛ばすパーティクル / DOM に貼る装飾。タップ非対象。
- prefix 衝突回避: ボタンは `btn_note_*` `btn_set_*` `btn_speed_*` `btn_ui_*`、装飾粒子は `spark_*` `deco_*`。

## ゴール
子供向け音遊びアプリ「音タッチ」のボタン 23 個（楽器 8 + 音色 6 + 速度 3 + UI 制御 6）を、**2 枚のアセットシート（Sheet 1a / Sheet 1b）**に分けて発注する。**baseline スタイル = ステッチ風円ボタン（クリーム/淡色のベタ背景 + 同色系の破線リング縁取り）＋ 中央に楽器/音ごとの個別の可愛いイラスト（ツヤあるキャラ的塗り）**。木目・木製質感は引き続き NG（音色を想起させてしまうため）、抽象音符（♪/♬・幾何記号だけ）も NG（「下手に音符にすると違いがわからなくなって覚えにくい」というユーザー判断）。アイコンとボタン土台は一体で描き、後で切り出して `assets/images/oto/buttons/` に配置する。

**【方針反転 2026-05-14】** Sheet 1a の「デザイン言語 4〜5 案提案フェーズ」（Phase1）は **完了済**。Codex 提案 A〜E ではなく、ユーザーが別ルートで Codex に発注した **ステッチ風円ボタン + 中央個別イラスト** 48 枚セットから 21 枚を選定して配置したものを **採用**。当初の「ステッチ・布製・刺繍は NG / 抽象モチーフのみ」という Phase1 制約は反転し、上記 baseline が音タッチ全ボタンの確定スタイルとなった（詳細は memory `feedback-oto-visual-direction` 2026-05-14 反転版）。

## スタイル方針（両シート共通 / 2026-05-14 反転反映版）

### DO
- ポップ、明るい、彩度高め、子供向けゲーム UI 寄り
- **ステッチ風 / 縫い目縁取りは OK・推奨**（クリーム/淡色のベタ背景 + 同色系の破線リング縁取り = 採用 baseline）
- **中央モチーフは楽器/音ごとに別の可愛いイラスト**（ツヤあるキャラ的塗り、🌸🦋⭐🍀🎈🍎🐟🌈系の具体物 OK）
- 光沢・ハイライト・ソフトグラデ・薄い内側グローは補助オプションとして併用可
- 円形・ピル型の土台、フラット〜セミフラット
- ボタン本体に**内包**ドロップシャドウ可（セル外への影出しは禁止）。Batch2 のパーティクル素材を後から合成する場合は**影なしで表示**できる前提なので、影は強くしすぎない
- 子供が「押せる」と直感する立体感（軽い 3D ボタン風）

### DON'T
- **木目・木製質感**（マリンバ/カリンバ/木琴等の音色を想起させるので NG）
- **抽象音符のみのモチーフ**（♪/♬ や波紋・幾何記号だけ。音ごとの違いがわからず子供が覚えにくいので NG。中央は具体イラストで識別させる）
- 全体が水彩でべったり塗りつぶされる絵本水彩 UI（ステッチ風円ボタン + キャラ塗りは OK だが、画面全体水彩は別問題で NG）
- くすみ色・アースカラー
- ポノ（クマ）を各ボタンに描き込む（タイトルロゴ専用）
- **アクティブグロー版を同シート内に作らない**（押下/アクティブ状態は CSS で表現する。Codex は通常状態のみ納品）
- 空きセルは**完全透過**で残す（背景・ガイド線・ロゴ等を入れない）

> 旧 DON'T で挙げていた「ステッチ・刺繍・布素材・フェルト・紙・ヴィンテージ」は **反転して OK** となった。「ポノを各ボタンに描き込まない」「木目 NG」だけは旧方針から継続。

## 共通スタイル基準 (Sheet 1b / Batch2 で適用 / 2026-05-14 反転反映版)

本セクションは Sheet 1b・**Batch2 (`batch2-particles.md`)** から参照される正本。これらでは以下の数値ガイドラインを統一すること。**Sheet 1a の Phase2 (最終発注) は廃止**（Phase1 完了後、別ルートのステッチ風円ボタン採用で決着済 = 後述「採用フロー」参照）。

> ⚠️ Sheet 1a Phase1 / Phase2 という 2-phase 運用自体は完了済。残る Sheet 1b と Batch2 への発注時は本基準に従う。

- **baseline スタイル**: **ステッチ風円ボタン（クリーム/淡色のベタ背景 + 同色系の破線リング縁取り）+ 中央に個別の可愛いイラスト** を Sheet 1b・Batch2 でも踏襲し、Sheet 1a と統一感を出す
- **ハイライト方向**: **左上 45°** から光が当たる前提で描く（Sheet 1b・Batch2 で統一）
- **縁取り**: **同色系の破線リング（ステッチ風）を推奨**。木目・木製質感は NG
- **内包ドロップシャドウ**: 不透明度 **25-35%** / ぼかし半径 **6-10px** / オフセット **下方 4-6px**（セル外に影を出さない）
- **ジェル質感 / グロー**: 補助オプション。高彩度ベースカラー **±20%** の範囲で **1〜2 層** の発光（内側 or 外周ソフトグロー）を必要に応じて重ねる
- **線アクセント**: 入れる場合は白 or 高明度の単色、太さ **2-3px**（256px 描画時の基準。Sheet 1b の 300×340 セルでは比例して 2.5-3.5px 程度）
- **角丸**: 外形が円でない場合の角丸は半径 **24px 前後**（256px 描画時の基準）

## Sheet 1a — 楽器ボタン（メイン操作 / 一番大きく綺麗に）

### シート仕様

#### bbox 機械抽出公式 / アンカー / safe area (Sheet 1a)
- **セル左上座標公式**: `x = (col-1) * cellW + paddingX`, `y = (row-1) * cellH + paddingY`
- **セル寸法表**:

  | パラメータ | 値 |
  |---|---|
  | cellW | 256 |
  | cellH | 256 |
  | paddingX | 18 |
  | paddingY | 18 |

- **描画アンカー**: 全セル **セル中心揃え** (`cellCenter = (x + cellW/2, y + cellH/2)`)。被写体は中心から左右上下対称に配置
- **safe area**: padding を除いた `cellW - 2*paddingX` × `cellH - 2*paddingY` = **220 × 220 px** を「safe area」と呼び、被写体はこの中に収める
