# ポノの海底図鑑 Stage / Zukan Spec

> 旧称「海底アルバム」は、以後「海底図鑑」へ統一する。
> UI 表示、仕様書、ユーザー向け文言はすべて「図鑑」を正とする。
> 既存コードや素材名に残る `Album` は移行前の内部名として扱い、今後の改修で `Zukan` 系へ寄せる。

## 目的

「ポノの海底図鑑」は、潜水艦で海を進み、生き物へエサを届けながら観察し、図鑑を完成させる 2D 横スクロール探索ゲーム。

主目的は次の 3 つ。

- 何度も遊んで図鑑コンプリートを目指せること。
- 魚、甲殻類、貝、クラゲ、古代生物、深海生物などの知識を自然に得られること。
- ステージごとに浅瀬、サンゴ、海藻の森、外洋、深海、古代海底神殿のように環境が変わること。

## Core Loop

1. ステージを選ぶ。
2. 潜水艦を動かして海中を探索する。
3. 生き物は最初、潜水艦を「住処をこわしに来た相手」と誤解し、泡、砂けむり、水流などで身を守ろうとする。
4. 通常エサ弾とぽよんエサ弾を使い分け、生き物へエサを届ける。
5. 生き物が `normal` から `eating`、満腹後に `happy` へ変化する。
6. 誤解が解けた生き物は、お礼、豆知識、宝物のヒントを一言だけ残して去る。
7. 初回満腹時に図鑑へ登録される。
8. 登録時に短いバナーを表示し、Gold / 報酬を得る。
9. 図鑑で「すむところ」「食べもの」「からだのひみつ」を読む。
10. 未発見、レア、ボス登録を目指して再挑戦する。

## Controls

- 移動: マウス / タッチドラッグ / キーボード。
- 通常エサ弾: 左クリック / 1 本指押しっぱなし / Space / Z。
- ぽよんエサ弾: 右クリック / 2 本指 / Shift / X。
- 通常エサ弾とぽよんエサ弾は、Unity 版では左ボタン押しっぱなし中に同時発射する現在仕様を許容する。
- ボム単独入力は、後で難易度調整が必要になった時に復活できるよう `WeaponController` 側で分離しやすくしておく。

## Weapon Growth

| Power | 通常エサ弾 | ぽよんエサ弾 | 補足 |
|---|---|---|---|
| 1 | 1 方向 | 1 個 | Stage 1 開始状態 |
| 2 | 2 方向 | 1 個 | 浅瀬の上下差に対応 |
| 3 | 3 方向 | 2 個 | 底にいる生き物へ届きやすくする |
| 4 | 3 方向、発射間隔短縮 | 2 個、爆風拡大 | 終盤とボス向け |

## Encounter Tone

生き物は倒す敵ではなく、最初だけ誤解している相手として扱う。

- 誤解: 潜水艦を「住処をこわしに来たもの」「食べものを奪いに来たもの」と思っている。
- 防衛反応: 泡、砂けむり、水流、威嚇光など。弾や攻撃ではなく、生き物らしい反応にする。
- 解決: エサを食べて満腹になると誤解が解け、`happy` 表情になる。
- 退場: お礼や短いヒントを一言だけ出して、上方向へ軽く退場する。
- 宝探し: 一部の生き物は、仲良くなったあとに宝箱、隠し通路、次に探す生き物のヒントを言う。

会話は短く、子どもが一瞬で読める長さにする。
吹き出しは楕円ではなく、読みやすい角丸長方形を基本にする。文字はプレイ中に読めるサイズを優先し、長い文は 2 行に折り返す。

| 状態 | 目的 | 例 |
|---|---|---|
| 警戒 | 最初は敵対しているように見せる | 「ここはぼくのすみかだよ」「こわしに来たの？」 |
| 食べ中 | 勘違いが揺らぐ | 「あれ、これ食べもの？」 |
| 満腹 | 仲直りと報酬 | 「ありがとう、おいしかった！」 |
| ヒント | 探索の導線 | 「左の岩かげ、きらっとしたよ」 |

## Stage Detail

### Stage 1: しおだまりと浅瀬

狙い: 一番明るく、親しみやすい導入ステージ。底にいる生き物へぽよん弾を使う理由を自然に教える。

出現生物:

| ID | 表示名 | すむところ | 食べもの | からだのひみつ |
|---|---|---|---|---|
| hermit_crab | ヤドカリ | 浅い海の岩場や貝殻の近く | 小さな有機物、食べ残し、小さな生き物 | 貝殻を背負って体を守る |
| shrimp | エビ | 岩陰、浅い海、海藻の近く | 小さなプランクトン、有機物 | 長いひげでまわりを探る |
| sea_star | ヒトデ | 岩場や砂地 | 貝などを食べる種類がいる | 星形の体で、ゆっくり動く |
| tidepool_goby | ハゼ | 浅い海の底や砂地 | 小さな虫、プランクトン | 底の近くで止まるのが得意 |
| sea_anemone | イソギンチャク | 岩にくっつく浅い海 | 触手にふれた小さな生き物 | 花のように見えるが動物 |
| horseshoe_crab | カブトガニ | 浅い海や砂地 | 小さな貝や虫のなかま | とても古い時代から形が大きく変わらない |

実装メモ:

- カブトガニは Stage 1 のボスに固定する。
- カブトガニはステージ開始時には出さず、現在のプレイで通常生物 5 種を全員満足させた後に `WARNING` を出してから出現する。
- ボス出現判定は図鑑の登録済み状態ではなく、そのプレイ中に満足させた通常生物 ID で判定する。
- ボス準備に入ったら通常生物は全て消し、通常生物の再スポーンも止める。
- WARNING 後、ボスがいる位置までカメラを強制スクロールする。ボスが全身見えてスクロールが止まってから、ボスのセリフを出す。
- ボスセリフが終わるまではボス行動を開始しない。
- ボス前 WARNING では、カブトガニが「ここをこわしに来たのか？」と誤解していることを示す。
- カブトガニの HP は通常生物より高く、現在は 30 を基準にする。
- ボス出現中は画面下部にボス HP ゲージを表示する。
- カブトガニの大技は「突進」と「しっぽ水流ビーム」の 2 種類に分ける。同時には出さない。
- 突進は、一瞬光る予兆のあと左方向へ速く走り、技が終わってからゆっくり元の位置へ戻る。
- しっぽ水流ビームは、突進とは別のタイミングで体を反対向きにし、水色に光る予兆のあと、しっぽ側からまっすぐ伸びる強い水流を出す。レーザーではなく、防衛反応として扱う。
- しっぽ水流ビームは青白く太い直線表現にし、紫やピンク寄りに見えないようにする。
- 通常エサ弾 / ぽよんエサ弾は、カメラ外に出たら消える。画面外の生き物が、姿を見せる前に満腹にならないようにする。
- プレイヤーと生物の接触は無効。エサ弾だけが当たる。
- 満腹後は `happy` 表情で上方向へ軽く退場し、同じスポーン枠ではコインを二重に落とさない。

Stage 1 会話案:

| 生物 | 警戒 | 満腹後 |
|---|---|---|
| ヤドカリ | 「この貝がらはぼくのだよ！」 | 「おいしかった。岩かげも見てみて」 |
| エビ | 「びっくりした！こっちに来ないで」 | 「ありがとう。すなけむりでごめんね」 |
| ヒトデ | 「ここでじっとしてるだけだよ」 | 「ごちそうさま。ゆっくり進むのも大事だよ」 |
| ハゼ | 「ここに来ても何もないぞ」 | 「左のほう、きらっとしたよ」 |
| イソギンチャク | 「さわるとあぶないよ」 | 「食べものだったんだね。ありがとう」 |
| カブトガニ | 「ここをこわしに来たのか？」 | 「勘違いしていたよ。海底にはまだ宝がある」 |

### Stage 2: サンゴの迷路

狙い: 隠れ家、共生、色の意味を扱う。サンゴのすき間を進むため、背景と前景の多重スクロールを強める。

図鑑対象候補:

- クマノミ: イソギンチャクと一緒に暮らす。
- ナンヨウハギ: サンゴ礁に住む青い魚。
- チョウチョウウオ: うすい体でサンゴの間を泳ぐ。
- タツノオトシゴ: 海藻やサンゴにしっぽでつかまる。
- クリーナーシュリンプ: 魚の体を掃除する。
- ボス: オウムガイ。古代的な雰囲気を持つ巻き貝のなかま。

### Stage 3: 海藻の森

狙い: 海藻が隠れ家になること、擬態やゆらぎを見せる。

図鑑対象候補:

- カニ: はさみを使う甲殻類。
- アワビ: 岩にくっついて暮らす貝。
- ウニ: とげで身を守る。
- 小魚の群れ: 群れで身を守る。
- イカ: 体の色を変える。
- ボス: ミズダコ。やわらかい体で岩陰に隠れる大型のタコ。

### Stage 4: 外洋の青い道

狙い: 広い海、群れ、大型生物、プランクトン食を扱う。横方向の開放感を出す。

図鑑対象候補:

- マンタ: 大きな体で泳ぎ、プランクトンを食べる。
- マンボウ: 外洋をゆっくり泳ぐ大きな魚。
- トビウオ: 水面近くから飛び出す魚。
- クラゲ: 水の流れにのって漂う。
- イワシの群れ: 群れで泳ぐ小魚。
- ボス: メガマウス。大きな口でプランクトンを食べる深海寄りのサメ。

### Stage 5: 夕暮れの中層

狙い: 暗くなり始めた海、発光、上下移動する生き物を扱う。

図鑑対象候補:

- ハダカイワシ: 光る器官を持つ小さな魚。
- ホウライエソ: 大きな口と鋭い歯を持つ深海魚。
- 細長い深海魚: 光が少ない海で暮らす体形を紹介。
- クラゲ: 暗い海で光って見える種類がいる。
- サルパ: 透明な体で漂うプランクトンのなかま。
- ボス: リュウグウノツカイ。長い体を持つ大型の深海魚。

### Stage 6: 深海の海底

狙い: 暗さ、低温、ゆっくりした動き、深海底の不思議さを出す。

図鑑対象候補:

- ダイオウグソクムシ: 深海の底で暮らす大きな甲殻類。
- 深海エビ: 暗い海で暮らすエビ。
- ナマコ: 海底の砂や有機物を食べる。
- アンコウ: 光る疑似餌で獲物を誘う種類がいる。
- タカアシガニ系: 長い脚を持つ大型のカニ。
- ボス: シーラカンス。古い時代から姿を残す魚として紹介。

### Stage 7: 古代海底神殿

狙い: 最終ステージとして伝説感を出す。ただし図鑑では「実在の生き物」と「伝説モチーフ」を分けて説明する。

図鑑対象候補:

- ダイオウイカ: 深い海にすむ巨大なイカ。
- 巨大クラゲ: 大きく育つクラゲのなかま。
- 古代魚風: 化石で知られる古い魚の形をモチーフにする。
- 深海甲殻類: 暗い海底に適応した甲殻類。
- ウミヘビ風: 長い体で泳ぐ生き物として紹介。
- ボス: クラーケン伝説。図鑑では「伝説の巨大生物。モデルは大きなイカやタコかもしれない」と説明する。

## 図鑑仕様

### 用語

- 正式名称: 図鑑
- 画面名: 海底図鑑
- 登録: 図鑑に登録
- 未登録: まだ見つけていない
- 旧称: アルバム

ユーザー向け文言では「アルバム」を使わない。既存の `AlbumManager`、`AlbumUI`、`album_registration_banner` などは今後 `ZukanManager`、`ZukanUI`、`zukan_registration_banner` へ移行する。

### 登録条件

- 生き物を初めて満腹にした時点で登録。
- 通常生物、ボスとも登録対象。
- 既に登録済みの場合は重複登録しない。
- 既に登録済みでも Gold / コインの再取得を許すかはステージ経済側で制御する。
- 同じスポーン枠での二重報酬は不可。

### 図鑑データ項目

`CreatureData` または今後の `ZukanEntryData` は、最低限次の項目を持つ。

| 項目 | 用途 |
|---|---|
| id | 保存用の一意 ID |
| displayName | 表示名 |
| stageId | 登場ステージ |
| category | 魚、甲殻類、貝、クラゲ、古代生物、伝説など |
| rarity | normal / rare / boss |
| habitat | すむところ |
| food | 食べもの |
| secret | からだのひみつ |
| childFact | 子ども向けの短い豆知識 |
| sprites | normal / eating / happy |
| silhouetteSprite | 未登録表示用。未指定なら normal の黒塗り |
| isBoss | ボスかどうか |

### UI 基本

現在の Unity 版では、プレイ中 UI は軽量に保つ。

- 表示: `図鑑 0/6`
- 次の未登録生物を 1 体だけシルエット表示。
- 新規登録時だけ `○○が図鑑に登録されたよ` のバナーを出す。
- 詳細リストは後続フェーズで追加。

後続の図鑑画面では次を実装する。

- ステージ別タブ。
- 登録済みカード。
- 未登録シルエット。
- 詳細カード。
- ボス枠。
- 全体達成率。
- ステージ別コンプリート印。

### 詳細カード

詳細カードの表示順は固定する。

1. 名前
2. すむところ
3. 食べもの
4. からだのひみつ
5. ひとこと豆知識

文体は短く、子ども向けにする。

例:

```text
名前: カブトガニ
すむところ: 浅い海や砂地
食べもの: 小さな貝や虫のなかま
からだのひみつ: とても古い時代から形が大きく変わらない
ひとこと: 生きた化石と呼ばれることがあるよ
```

### 保存仕様

- 保存キーは Unity 版では `PonoSubmarine.Zukan.UnlockedIds` のように、図鑑名を含める。
- 旧 `Album` 保存キーが存在する場合は、初回起動時に `Zukan` 保存へ移行する。
- 移行後も旧キーを即削除せず、一定期間は読み取り fallback として残す。

### Completion

図鑑の達成段階は 3 層にする。

| 達成 | 条件 | 表示 |
|---|---|---|
| Stage Complete | そのステージの通常生物を全登録 | ステージに小さな印 |
| Boss Complete | ボス登録 | ボス枠に印 |
| Full Complete | 全ステージ全登録 | 海底図鑑コンプリート |

Stage 1 の達成数は `6/6`。全 7 ステージで 6 体ずつなら最小 `42` 種。レア枠を追加する場合は `42 + rare` とする。

## Asset Policy

- 生物は `normal` / `eating` / `happy` の 3 表情を基本とする。
- 既存のアルファ付き PNG がある場合は必ずそれを使う。
- 潜水艦は既存 `Submarine_003.png` を使い、勝手に作り直さない。
- 泡、水流、ヒットリング、砂けむり、きらめきは画像生成ではなく Particle / procedural sprite / shader で作る。
- 新規画像生成が必要な場合は GPT Image 2 のみ。
- 生成画像は raw として保存し、必要ならユーザーがアルファ付きにする。

## Rename Checklist

今後「アルバム」から「図鑑」へ寄せる時のチェック。

- UI 文言: `アルバム` -> `図鑑`
- クラス名: `AlbumManager` -> `ZukanManager`
- クラス名: `AlbumUI` -> `ZukanUI`
- メソッド名: `UnlockCard` / `RegisterAlbum` 系 -> `UnlockZukanEntry` / `RegisterZukanEntry`
- 保存キー: `Album` を含むキー -> `Zukan` を含むキー
- 画像名: `album_registration_banner` -> `zukan_registration_banner`
- Scene 内 GameObject 名: `AlbumUI` -> `ZukanUI`
- Data Asset 名: `CreatureData` は維持可。図鑑専用に分けるなら `ZukanEntryData`

既存実装を一度にリネームすると参照切れが起きやすいので、挙動が安定している間は文言優先、内部名は別 batch で段階的に変更する。
