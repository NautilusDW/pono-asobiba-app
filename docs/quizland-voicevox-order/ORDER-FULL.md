- リスの女の子、フクロウ博士のアシスタント役

## 出力仕様
- 形式: wav（48kHz / 16bit / モノラル）
- 命名規則: `q{NNN}_{q|a|b|c|d}.wav`（NNN は 3桁ゼロパディング、Q01〜Q180 を q001〜q180、補足 Q181 を q181 として扱う）
  - Q01〜Q99 → q001〜q099, Q100〜Q181 → q100〜q181
  - パート: `q` (問題文) / `a` / `b` / `c` / `d` (選択肢)
  - Q160 補足版: `q160_a_alt.wav` / `q160_b_alt.wav`
- 出力ディレクトリ: 納品時に別途指示（ひとまず `audio/quizland/` を想定）

## 読み方ルール

### ① 色名の英語キー → 日本語（order_color）
order_color カテゴリの選択肢は内部データでは英語キーで持っているが、画面表示・音声ともに日本語表記:

| 英語キー | 日本語（speech） |
|---|---|
| red | あか |
| blue | あお |
| yellow | きいろ |
| green | みどり |
| pink | ピンク |
| orange | オレンジ |
| purple | むらさき |
| cyan | みずいろ |

### ② count_total（Q24〜Q47）の選択肢の読み方
- 画面表示は「2つ」「3つ」…「9つ」「10こ」（10だけ「こ」を使う）
- speech は **和語数詞**: ひとつ / ふたつ / みっつ / よっつ / いつつ / むっつ / ななつ / やっつ / ここのつ / **じゅっこ**
  - 注: 画面「10こ」に対して speech は「じゅっこ」(訓読み) で揃える

### ③ number_sequence（Q169〜Q180）の選択肢の読み方
- 画面表示は裸数字「1」「2」「3」…「10」
- speech は **音読み**（数字読み）: いち / に / さん / よん / ご / ろく / なな / はち / きゅう / じゅう
  - count_total の和語読みと使い分ける（文脈が「次の数」なので音読みが自然）
- 問題文の読み:
  - 「1の つぎは？」→ speech「いちの つぎは？」
  - 「2の つぎは？」→ speech「にの つぎは？」
  - 「2と 4の あいだは？」→ speech「にと よんの あいだは？」
  - 「9の つぎは？」→ speech「きゅうの つぎは？」
  - 「10の まえは？」→ speech「じゅうの まえは？」

### ④ opposite（Q95〜Q118）の「」(カギ括弧)
- 画面表示は「「おおきい」の はんたいは？」のようにカギ括弧付き
- speech は **カギ括弧を除去**: 「おおきい の はんたいは？」と読ませる

### ⑤ shape_name 等の助数詞付き選択肢の正規化
- 画面「3こ」→ speech「さんこ」
- 画面「5ほん」→ speech「ごほん」
- 画面「6ぽん」→ speech「ろっぽん」
- 画面「7しょく」→ speech「ななしょく」
- 数字を speech 側でカナ化して読み崩れを防ぐ
- 「0こ（ない）」→ speech「ぜろこ」（括弧の「ない」は無視）

### ⑥ Q160「はい(肺)」/「い(胃)」の同音異義リスク
- Q160 選択肢A「はい」、選択肢B「い」は臓器名で同音異義のリスクあり
- 対応: **両案併記** にする
  - 通常版 (`q160_a.wav`): 「はい」
  - 補足版 (`q160_a_alt.wav`): 「ぞうきの はい」
  - 通常版 (`q160_b.wav`): 「い」
  - 補足版 (`q160_b_alt.wav`): 「いぶくろの い」
- 試聴後にどちらを採用するかユーザー判断

### ⑦ 句読点・記号
- 文末の「？」は残してOK（VOICEPEAK が自然に処理する）
- 「、」「・」もそのまま
- 半角スペースは VOICEPEAK の入力でも自然なポーズになるので残す
