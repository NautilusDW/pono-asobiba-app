# Implementation Plan: ボウリング爽快感リニューアル v2

## Research Summary

3つのリサーチから抽出した核心:

### 爽快感の公式
```
爽快感 = (見た目の派手さ / 操作の簡単さ) × フィードバックの重層性 × 連鎖の深さ
```

### 最重要原則
1. **Small Input → Big Spectacle** — 1タップで画面が大きく反応する
2. **多層フィードバック** — 1つの衝突に対して視覚+音+揺れ+スローを同時に
3. **連鎖反応** — 1アクションが次のアクションを自動で引き起こす
4. **おもちゃ設計** — ゲームオーバーなし、勝ち負けなし、触るだけで楽しい
5. **Permanence** — 壊したものが残る（残骸、跡）

### 現状の問題点
- ボールがピンに当たっても**ピンが消えるだけ**で地味
- **連鎖がない** — ボールがピンを壊すだけ、ピン同士の連鎖がない
- **フィードバックが単層** — 音だけ、視覚が弱い
- **squash/stretch なし** — 衝突の重量感がない
- **ヒットストップなし** — 衝突の瞬間が流れてしまう

## Implementation Steps (優先順)

### Step 1: ピン破壊を「爆発」に変える
**現状**: ピンが消えるだけ
**改善**: ピンが壊れたら破片が飛び散り、その破片が他のピンに当たって連鎖破壊

```
ピン破壊時:
  - ピンの位置から8-12個の小さな破片(debris)を生成
  - 破片にも物理判定あり（ピンに当たると速度を加算）
  - 破片が当たったピンも破壊閾値を超えたら連鎖破壊
  - 連鎖するほどパーティクル増加＋音のピッチ上昇
```

**期待効果**: 1球で画面全体が連鎖的に爆発する「カタルシス」

### Step 2: ヒットストップ（フリーズフレーム）
**現状**: 衝突が流れて見える
**改善**: 強い衝突の瞬間、物理を3-5フレーム停止

```
衝突強度 > 閾値 のとき:
  - physicsTimeScale = 0（物理停止）
  - パーティクルとUIは動き続ける
  - 3-5フレーム後に徐々に復帰（0→0.3→0.7→1.0）
```

**期待効果**: 「ドン！」という衝撃の瞬間が際立つ

### Step 3: スクリーンシェイク改善（Perlinノイズ）
**現状**: 軽い揺れ
**改善**: trauma値ベースの有機的な揺れ

```
- trauma値（0-1）を衝突強度で加算、毎フレーム減衰
- shake = trauma^2（急激に始まり滑らかに収束）
- X/Y offset = maxShake * shake * sin(time * freq + seed)
- 最大振幅: 8px、回転: なし（2Dなので）
```

### Step 4: squash & stretch
**現状**: ボール/ピンは常に同じ形
**改善**: 速度に応じて進行方向に伸び、衝突時に潰れる

```
描画時:
  speed = sqrt(vx²+vy²)
  stretch = 1 + min(speed * 0.02, 0.3)
  angle = atan2(vy, vx)
  ctx.save()
  ctx.translate(x, y)
  ctx.rotate(angle)
  ctx.scale(stretch, 1/stretch)  // 体積保存
  ctx.drawImage(...)
  ctx.restore()

衝突時:
  squashTimer = 6  // 6フレームかけて復帰
  squashAmount = min(衝突強度 * 0.1, 0.4)
```

**期待効果**: 物体に「重量感」と「弾力感」が生まれる

### Step 5: 音のピッチ連鎖上昇
**現状**: ランダムなピッチ変化
**改善**: 連鎖中はピッチが上がり続ける（ドレミファ...）

```
chainPitch = 1.0
連鎖が続くたびに:
  chainPitch *= 1.08（半音ずつ上昇）
  playBuf(bonkBuf, vol, chainPitch)
  // 最大2.0まで
連鎖が途切れたら:
  chainPitch = 1.0にリセット
```

**期待効果**: 連鎖が続くほど音が気持ちよく上がっていく（パズルゲームの定番）

### Step 6: 衝撃波メカニクス
**新機能**: ピンが壊れた時、周囲に衝撃波（見えない力）を発生

```
ピン破壊時:
  半径100px以内の全ボディに外向きの力を加える
  力 = 破壊した衝突の強度 × (1 - distance/100)
  → 近くのピンが吹き飛ぶ → さらに連鎖
```

**期待効果**: ドミノ倒し的な連鎖反応。1球で画面全体が動く

### Step 7: タッチの改良 — 長押しでチャージ砲
**現状**: タップで1球、ドラッグで連射
**追加**: 長押し（500ms以上）で巨大チャージボールを発射

```
pointerdown → タイマー開始、チャージエフェクト表示
  - 指の周りにリング状のチャージゲージ
  - 500ms: 通常の2倍サイズ
  - 1000ms: 3倍サイズ + 画面が少し暗くなる演出
pointerup → チャージ量に応じた巨大ボールを発射
  - 大きいボール = 大きい質量 = 大きい破壊
```

**期待効果**:
- 3歳: 長押しで「すごいの出た！」という驚き
- 大人: チャージのタイミングを計る楽しさ

### Step 8: 背景の反応
**現状**: 静的な緑の背景
**改善**: 衝突の強度で背景色が微妙にフラッシュ

```
大きな衝突/連鎖時:
  bgFlash = 0.3（白方向に）
  毎フレーム bgFlash *= 0.9 で減衰
  背景色 = lerp(#1e6b40, #ffffff, bgFlash)
```

**期待効果**: 画面全体が反応している感覚

### Step 9: インパクトリング（衝突点エフェクト）
**新機能**: 衝突点から広がるリング

```
衝突時:
  rings.push({x, y, radius: 5, maxRadius: 50 + strength*20, alpha: 0.6})
毎フレーム:
  ring.radius += 3
  ring.alpha *= 0.92
  ctx.arc(ring.x, ring.y, ring.radius, ...)
  ctx.strokeStyle = rgba(255,255,255, ring.alpha)
```

**期待効果**: 衝突の「波紋」が可視化される

### Step 10: ピンの湧き方をもっとダイナミックに
**現状**: 上からゆっくり降る
**改善**: 複数パターンをランダムで

```
パターン:
A. 上から雨のように（現状）
B. 画面端から横に流れる
C. 画面中央から放射状に湧き出す
D. 一気に大量にドサッと降る（ボーナスウェーブ）
  - 30秒に1回、画面上部から20個まとめて落下
  - 予告: 画面上部が赤く光る → ドサッ
```

**期待効果**: 飽きない、パターンの変化で新鮮さ

## Risks and Mitigation
| Risk | Mitigation |
|------|------------|
| 破片でbody数爆発→FPS低下 | 破片に短い寿命(1s)＋MAX_BODIES制限 |
| ヒットストップが連続で操作不能感 | 最大0.1s、連続時は間引き |
| squash/stretchの計算コスト | save/restore最小限、速度0のbodyはスキップ |
| 衝撃波で無限連鎖→フリーズ | 1フレームあたりの連鎖数上限(10) |
| チャージ中に誤操作 | 300ms以下はチャージなし扱い |
