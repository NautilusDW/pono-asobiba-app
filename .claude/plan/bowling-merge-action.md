# Implementation Plan: マージアクション物理おもちゃ

## コンセプト
「タップでボール発射 + ピンが降ってくる」現行システムに、
**ピン同士が衝突したら合体して大きくなる**マージ要素を追加。
最大サイズまで育てると大爆発 → 周囲を巻き込む連鎖。

**スイカゲームとの決定的な違い:**
- 能動的にボールを撃ってピンをぶつけ合う（スイカ: 受動的に落とす）
- ピンが常に降ってくる動的な画面（スイカ: 静的な容器）
- ゲームオーバーなし、エンドレス（スイカ: 容器溢れで終了）
- マージは結果であって目的ではない（スイカ: マージが主目的）

## ピン進化チェーン（5段階）

ねこ+ねこ → **大爆発**（消滅 + 衝撃波 + 大量パーティクル + 大スコア）

## ゲームループ変更

### 現行（変更なし）
- ピンが上から降ってくる（tier 0〜1のみ自然湧き）
- タップでボール発射
- ドラッグで連射
- ボールは壁で跳ね返る
- ボールは時間経過で消える

### 新規追加
1. **ボールはピンを破壊しない** → 弾き飛ばすだけ
2. **同じtierのピン同士が接触 → 合体**
   - 2体を削除、中間点に次tierのピンを1体生成
   - 合体エフェクト（パーティクル + 音のピッチ上昇 + ポップアップ）
3. **ねこ同士の合体 → 大爆発**
   - 半径120px以内の全ボディに外向きの力
   - 爆発でさらにピンがぶつかり合い連鎖マージ
   - 大量パーティクル + 画面揺れ + 高スコア
4. **コンボ**: 1秒以内の連続マージでコンボ倍率
5. **音のピッチ連鎖**: マージするたびにピッチが上がる（ドレミ...）

## 核心の面白さ

### 3歳の遊び方
- バンバンタップしてボールを出す
- ボールがピンにぶつかってピンが飛ぶ → 楽しい
- 飛んだピンが別のピンにぶつかって光って大きくなる → すごい！
- 大きいのがぶつかると爆発する → わーい！
- 何も考えなくても連鎖が起きる

### 大人の遊び方
- 同じ種類のピンが近くにいるのを見つける
- ボールを狙って撃って同種ピン同士をぶつける
- 連鎖を意図的に起こしてコンボスコアを稼ぐ
- ねこまで育てて大爆発を狙う
- ハイスコアを更新する

## 実装ステップ

### Step 1: ピンにtierを持たせる
- body に `tier` プロパティ追加（0-4）
- tier別の半径・質量・画像の定義テーブル
- 各tierに対応するプリレンダーcanvasを各サイズで生成
- spawnPin() は tier 0 または 1 をランダムに生成

```js
const TIERS = [
  { img: 0,  r: 18, mass: 0.5 },  // うさぎ
  { img: 3,  r: 24, mass: 1.0 },  // パンダ
  { img: 6,  r: 32, mass: 2.0 },  // ハリネズミ
  { img: 9,  r: 42, mass: 4.0 },  // ペンギン
  { img: 12, r: 54, mass: 8.0 },  // ねこ
];
```

### Step 2: ピン同士の衝突でマージ判定
- physicsStep内の衝突ループで:
  - 両方ともtype==='pin' かつ tier が同じ → マージフラグ
  - マージはphysicsStep外で処理（物理ステップ中のbody追加/削除を避ける）
  - 1フレームあたりのマージ数を制限（最大5件、連鎖爆発防止）

```js
// physicsStep後に実行
const merges = []; // [{a, b, tier, x, y}]
// 衝突ループ内で:
if (a.type==='pin' && b.type==='pin' && a.tier===b.tier && !a.merging && !b.merging) {
  a.merging = b.merging = true;
  merges.push({a, b, tier: a.tier, x: (a.x+b.x)/2, y: (a.y+b.y)/2});
}
// マージ処理:
for (const m of merges.slice(0, 5)) {
  m.a.dead = true;
  m.b.dead = true;
  if (m.tier < 4) {
    // 次tierのピンを生成
    const t = TIERS[m.tier + 1];
    const nb = addBody(m.x, m.y, t.r, t.mass, 'pin');
    nb.tier = m.tier + 1;
    nb.imgIdx = t.img;
  } else {
    // tier4同士 → 大爆発
    triggerExplosion(m.x, m.y);
  }
  // エフェクト
  spawnParticles(m.x, m.y, 6 + m.tier * 4);
  spawnPopup(m.x, m.y - 20, TIER_NAMES[m.tier+1] || '💥');
  // 音（ピッチ上昇）
  mergePitch = Math.min(mergePitch * 1.12, 2.5);
  playBuf(popBuf, 0.4 + m.tier * 0.15, mergePitch);
  // スコア
  score += (m.tier + 1) * 10 * (1 + combo * 0.5);
  combo++; comboTimer = COMBO_WINDOW;
}
```

### Step 3: ボールでピンを破壊→弾き飛ばしに変更
- 現行: ボール-ピン衝突でpin.dead=true
- 変更: ボール-ピン衝突はphysics resolveのみ（弾くだけ）
- 強い衝突時のパーティクル・音は残す

```js
// 変更前:
if (preSpd > 3) { pin.dead = true; ... }
// 変更後:
if (preSpd > 3) {
  // ピンを破壊しない、弾き飛ばすだけ
  // 衝突エフェクトだけ出す
  triggerShake(strength / 15);
}
```

### Step 4: 大爆発（tier4マージ時）
```js
function triggerExplosion(cx, cy) {
  // パーティクル大量
  spawnParticles(cx, cy, 40);
  // 衝撃波: 半径150px以内のピンに外向きの力
  for (let i = 0; i < bodies.length; i++) {
    const b = bodies[i];
    if (b.dead) continue;
    const dx = b.x - cx, dy = b.y - cy;
    const dist = Math.sqrt(dx*dx + dy*dy);
    if (dist < 150 && dist > 1) {
      const force = (1 - dist/150) * 15;
      b.vx += (dx/dist) * force;
      b.vy += (dy/dist) * force;
    }
  }
  // 画面揺れ（大）
  triggerShake(3);
  // スコアボーナス
  score += 500;
  spawnPopup(cx, cy - 30, '💥 +500', '#FF4444');
  // 爆発音
  playBuf(comboBuf, 1.0, 0.6);
}
```

### Step 5: 音のピッチ連鎖
- `mergePitch` 変数: マージするたびに×1.12
- comboが途切れたら1.0にリセット
- 連鎖が続くとドレミファ...的に上がっていく

### Step 6: ビジュアル改善
- 各tierのピンをそのサイズに合わせてプリレンダー
  - pin_0.png → 36×36 canvas
  - pin_3.png → 48×48 canvas
  - ...
- 合体の瞬間: 白フラッシュ（1-2フレーム白い円を描画）
- 大きいピンほど存在感がある（影を大きく）

### Step 7: ピン湧きの調整
- tier 0（うさぎ）: 70%の確率で湧く
- tier 1（パンダ）: 30%の確率で湧く
- tier 2以上は自然湧きしない（合体でのみ生まれる）
- 画面上のピン数が15未満なら追加湧き

### Step 8: ジュース（演出追加）
- スクリーンシェイク: 合体tier×強度
- ヒットストップ: tier3以上の合体で3フレーム物理停止
- 合体リング: 合体地点から広がるリングエフェクト
- コンボUI: 「5コンボ！」表示
- ハイスコア: localStorage保存

## 技術的注意点

| 懸念 | 対策 |
|------|------|
| 合体でbody数が増減 → 物理ステップ中の配列変更 | マージ判定はphysicsStep外で実行 |
| 連鎖爆発で無限ループ | 1フレームあたりマージ上限5件 |
| 大きいピン(tier4, r=54)が多くなるとFPS低下 | tier4は爆発で消えるので自然に淘汰 |
| ピン画像の拡大でぼやける | tierごとに個別サイズでプリレンダー |
| 衝撃波で画面外に飛ぶ | wallBounceが既に機能 |
| ドラッグ連射＋大量ピンで衝突計算O(n²) | MAX_BODIES=80で制限 |
