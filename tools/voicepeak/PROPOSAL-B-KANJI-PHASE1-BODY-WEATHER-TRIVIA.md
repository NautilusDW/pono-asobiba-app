## ポリシー要約

| 観点 | 方針 |
|---|---|
| 最優先 | VOICEPEAK が正しく読めること |
| 数字 | 一〜十 は辞書登録済 → 全て漢字化 |
| 方向 | 左/右/前/次/間 は辞書登録済 → 漢字化 |

### body 留意点

- **行 14 「歯」 / 行 22 「肺」 / 行 23 「臓器の 肺」 単独発話**: 単独漢字 1 字 「歯」 「肺」 は VOICEPEAK では極端に短く 1 モーラで読まれる。 既存辞書 line 60 で `肺 ハイ 0 (8192;8193)` と音高指定があるので OK。 「歯」 は新規追加必須で `ハ, 1, 8193` 推奨 (頭高で 「ハ ↓」)
- **行 8/9 「みる」 「きく」**: 漢字化すると 「見る/聞く」。 「見る」 はミル, 1 / 「聞く」 はキク, 2 で安定。 辞書追加任意 (VOICEPEAK 標準辞書でも読める)
- **行 18 「血」 単独**: ひらがな 「ち」 では VOICEPEAK が 「ち」 として読まずスキップする恐れあり → 漢字 「血」 + 辞書登録 `チ, 1` 強推奨
- **行 15 「舌 (シタ)」 vs 既存 「した (シタ, 0)」**: 既存 line 35 は 「下」 の意味のシタ核 0。 「舌」 はシタ核 2 で別エントリが必要。 単漢字 「舌」 で別途登録
- **行 32 「お腹」**: 「腹 (ハラ)」 ではなく 「お腹 (オナカ)」 で登録。 漢字単独 「腹」 は使わない
- **行 19 「心ぞう」 表記**: 「心臓」 だと VOICEPEAK が 「ココロゾー」 等に揺れるリスク → 「心ぞう」 (混ぜ書き) が安全。 辞書には `心ぞう, シンゾー, 0` で登録
- 動物名・カタカナ語は body には登場しないので問題なし

## 全体留意点 (横断)

1. **「歯」 と 「葉」 は同読み (ハ, 1)** — VOICEPEAK 辞書では同一漢字 1 つにつき 1 エントリのため衝突しない。 同読み登録で運用 OK
2. **「目」 と 「芽」 は同読み (メ, 1)** — 同上、 衝突なし
3. **「雨」 の核は曖昧** — 既存 line 43 は核 0 (アメ平板=飴寄り)。 漢字 「雨」 は core 核 1 (頭高) で別エントリ推奨。 ただし 5-6 歳児にとっては平板でも違和感は小さいため、 試聴で気にならなければ既存核 0 流用も可
4. **動物名は全カタカナ維持** — VOICEPEAK 標準辞書が漢字より安定。 ねこ/うさぎ/ぞう/にわとり も慣例的にひらがな維持 (オリジナル CSV を尊重)
5. **オノマトペ・カタカナ語維持** — グー/ニャーニャー/ふわふわ/もや/つぶ/たてがみ などはそのまま
6. **「？」 (全角疑問符) と読点 「、」 のスペーシング** — 参考 CSV (number_sequence) では 「？」 直前にスペースなし、 文中の助詞には半角スペース挿入の事例あり。 本提案では既存スタイルを踏襲 (例: 「これは 体の どこ？」)
7. **混ぜ書き (kana+kanji) のスタイル統一** — 「心ぞう」 「地きゅう」 「皮ふ」 「雪ぐに」 の様に、 後半をひらがなに開く形式で読み崩れを回避
8. **読みのモーラ確認** — 「肺 (ハイ)」 「血 (チ)」 「葉 (ハ)」 等の短いモーラ単独語は、 音高指定 (例: `8192;8193`) を追加することで頭高がより安定 (既存 line 60 の `肺 ハイ 0 8192;8193` がベストプラクティス)
9. **試聴フローを必ず通す** — 本提案は理論ベース。 実装後に number_sequence と同じ Step 3 (試聴 + 微調整) を CSV ごとに必ず実行し、 不安定な語は辞書再調整
