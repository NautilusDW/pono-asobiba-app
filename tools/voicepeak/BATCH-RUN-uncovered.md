- プリセット **「女性4」** を選択 (くるみ確定話者、 [memory/feature_quizland_voicepeak_pivot_jyosei4.md](../../memory/feature_quizland_voicepeak_pivot_jyosei4.md))

> 本バッチ用に辞書追加が必要な語候補は 「## 辞書追加候補」 節参照。 試聴で読み崩れが顕著であれば別タスクで追加。

## Step 1: VOICEPEAK で CSV インポート

- **ファイル**: `tools/voicepeak/voicepeak_lines_unique_phase2_uncovered.csv`
  - 1 列目 = `女性4` (全 190 件統一済)
  - 2 列目 = speech (**漢字混じり化 + 句点付き**: phase1 phase2 ルール踏襲)
    - 漢字化: 確実に読める一意読み漢字のみ (例: 「右 / 左 / 上 / 速い / 軽い / 重い / 春 / 夏 / 秋 / 冬 / 津波 / 地震 / 犬 / 牛 / 豚」)
    - カナ維持: 1 モーラ漢字 (「め / て / は / い / け」)、 同形異義 (「はな = 鼻/花/端」 「あし = 足/脚」 「かみ = 髪/紙/神」 「くも = 雲/蜘蛛」 「ゾウ = 象/像」 「もよう = 模様」)、 常用外 (「にらむ / にわとり / ひげ / ほほ / ものさし」)、 既出方針継承 (「いい / かたい / かわいい / きつい / おなじ / こわい / ねむい / がら / つめ / がら / はえる」)
    - 数字 + 助数詞: body / trivia phase1 と統一 (「4本 / 6本 / 3本 / 10本 / 15個 / 20個 / 5個」 = アラビア + 漢字助数詞)
    - 数詞 + 助数詞 (shape_name): 漢数字 (「一個 / 二個 / 三個 / 四個 / 五個 / 六個」 = count_total phase1 と統一)
  - ヘッダなし、 UTF-8 (BOM なし)、 CRLF (= 190 行確定、 4235 bytes)
- **プリセット**: 「女性4」
- **サンプルレート**: 44100 Hz
- **命名規則**: 連番 `0, 1, 2, ...` / プレフィックスなし / サフィックスなし
  - → 出力: `0-uncovered.wav` 〜 `189-uncovered.wav` (190 ファイル)
- **出力先フォルダ**: `D:\AppDevelopment\pono-asobiba-app\tmp\quizland_NA\phase2_uncovered`

> 命名規則は他カテゴリ (count_total / number_sequence / shape_name / weather / opposite / body / trivia) と同じ。 Expand スクリプトは BaseName 先頭の数字で natural sort するため、 `0`/`00`/`001` のいずれでも OK。

## Step 2: 一括書き出し

VOICEPEAK GUI で全行選択 → 書き出し → Step 1 のフォルダへ保存。

→ `0-uncovered.wav` 〜 `189-uncovered.wav` (190 件) が生成される。

## Step 3: Expand スクリプト実行 (q番号への展開)

本バッチは cross-category 集約のため、 各 phase2 expand JSON に分岐して展開する必要がある。 5 カテゴリの phase2 expand JSON はキーがカナ、 order_color phase2 expand JSON はキーが漢字+句点付き、 という形式差異があるため、 既存の `Expand-VoicepeakUniqueWavs.ps1` を 6 回 (5 cat + order_color) 別々に呼ぶか、 新規スクリプトでまとめて展開する。

### 推奨手順 (案 A: 既存スクリプトを 6 回呼ぶ)

cross-cat dup 7 件 (= おなじ。/飛ぶ。/夕方。/真夜中。/4本。/6本。/煙。) は同一 unique wav から複数カテゴリの q### に展開される。 既存スクリプトは入力 unique CSV と expand JSON のキーマッチで動作するため、 集約 CSV (= キー漢字+句点付き) と 5 カテゴリ phase2 expand JSON (= キーカナ) は直接マッチしない。 そのため案 A では新規 unique CSV を 6 つに分割再生成する必要がある (= 手間)。

### 推奨手順 (案 B: 集約 detail JSON を使う新規展開)

`tools/voicepeak/voicepeak_unique_expand_phase2_uncovered_detail.json` には各 unique 語が展開される全 q### (= 「category:q###_x.wav」 形式) が記録済。 これを使って新規スクリプトでまとめて展開する。

```powershell
cd d:\AppDevelopment\pono-asobiba-app
# (新規スクリプト未作成 — 必要なら別タスクで作成)
# 暫定: voicepeak_unique_expand_phase2_uncovered_detail.json を読んで各 unique wav (190 件) を該当 q### に複製
```

### 暫定手順 (検証優先)

1. Step 2 で 190 件の wav を `tmp/quizland_NA/phase2_uncovered/` に生成
2. wav 試聴 → 漢字化の読み崩れチェック (= 後述 「試聴ポイント」)
3. 読み崩れ修正 (辞書追加 or CSV 再編集) を経てから q### 展開スクリプトを作成・実行 (= 別タスク)

## Step 4: 試聴 + 微調整

展開後の `q###_*.wav` 群を順次試聴。 読み崩れがあれば:

1. `tools/voicepeak/voicepeak_user_dict.csv` を更新
2. `powershell -ExecutionPolicy Bypass -File tools/voicepeak/Convert-VoicepeakUserDictCsvToVdc2.ps1` で .vdc2 化
3. VOICEPEAK GUI でユーザー辞書を再インポート
4. 集約 CSV を VOICEPEAK で再書き出し → 展開スクリプト再実行で全 q### に自動反映

## 試聴ポイント (カテゴリ横断)

> **漢字混じり化 + 句点付き**。 phase1 で確立した 3 ルール (漢字化 / カナ維持 / 句点) を踏襲。 試聴で読み崩れが顕著であれば辞書追加候補 (= 後述) を別タスクで実施。

| ポイント | 確認内容 |
|---|---|
| 数詞 + 助数詞 (個系) | 「一個 / 二個 / 三個 / 四個 / 五個 / 六個 / 15個 / 20個 / 5個」 が単一語として読まれるか。 「ヒトコ / ニコ / サンコ / ヨンコ / ゴコ / ロッコ」 などのアクセント核と各音の長さが count_total phase1 と整合しているか |
| 数字 + 助数詞 (本系) | 「3本 / 4本 / 6本 / 10本」 が単一語として読まれるか。 「サンボン / ヨンホン / ロッポン / ジュッポン」 が音便変化を保ったまま発音されているか |
| 反対語の形容詞・動詞 | 「赤い / 丸い / 速い / 大きい / 多い / 軽い / 明るい / 強い / 描く / 怒る / 安い / 食べる / 言う / 話す / 聞く / 読む」 が phase1 「小さい / つめたい / 柔らかい / 弱い / 笑う」 等と同レベルのアクセント精度で読まれるか |

| 色名 (新規) | 「青 / 紫 / 黒 / 白 / 砂 / 氷 / 水 / 石」 (trivia) が一意読み漢字として自然に読まれるか。 「青 (アオ, 核 1)」 「紫 (ムラサキ, 核 0 平板想定)」 のアクセント要確認 |
| みずいろ。 | order_color phase2 で唯一未収録の 「みずいろ。」 が 「水色」 漢字ではなく**意図的にカナ維持** (= phase1 order_color 8 色は漢字化済だが、 みずいろは「水色 → スイショク誤読リスク + 子供向け絵本的やわらかさ」 でカナ維持)。 アクセントは 「ミ↑ズイロ↑」 (核 0 平板想定) で素直に読まれるか |
| 句点による発音完了感 | 全 190 件で末尾「。」 が VOICEPEAK の自然な発音完了感 (= 末尾の voice trail / 文末感) を生むか。 phase1 body / trivia / order_color と同じレベルで安定しているか |
