# Pono LP Brand Kit

絵本『ありがとうって、うれしいね 〜くまのこポノのちいさなものがたり〜』(こだまのもり出版) の公式 LP「ポノのあそびば」(`index.html`) を、Claude Design 上でブランドに沿ったバリアント生成するための Brand Kit + プレビューシェル一式です。ヒーローセクションと「えほん×あそびば」セクションをトークン付きで切り出してあり、claude.ai/design 上で 3 案ずつ並列に生成・比較し、選んだものを実プロジェクトの `index.html` に反映する運用を想定しています。

---

## ブランドボイス

- 対象: 3〜6 歳の子ども＋その保護者
- トーン: やさしい / あたたかい / てづくり感 / 低圧 / 誘導しない
- 語り手: 「ネモ先生」名義、こだまのもり出版の丁寧語
- 配色トークン:
  - `--bg-milk: #FFF8F0` (ミルク背景)
  - `--primary: #F2915A` (くまオレンジ / アクセント)
  - `--text-brown: #5D4E37` (本文)
  ※ 詳細は `tokens/colors.html` を正本とする
- 書体: Zen Maru Gothic (700/900)、`clamp()` で可変、ひらがな多め
- 言い回し例: 「あそびば」「いりぐち」「みてね！」
- 装飾: 🌳🌱🎵 等の絵文字＋角丸 14〜28px ＋淡い影。広告的な原色・強い影は禁止

---

## LP の戦略

このサイトは **絵本『ありがとうって、うれしいね』の世界をやさしく届ける入口** として、「絵本そのもの」と「ポノのあそびば (Web ゲーム集)」をふたつの扉として並べる二段構成です。訪れた保護者には絵本に出会ってもらいやすく、子どもには「ポノの世界で遊んで → 楽しい → 絵本も読みたい」と感じてもらえる、ゆるやかな動線を目指します。

ヒーローでは「絵本ブランドとしての品格」を保ちつつ、「あそびばの入口」を視覚的に等価に並置します。「えほん×あそびば」セクションでは、絵本ジャケットと遊び場ビジュアルを **どちらが主役・脇役かを決めつけず、地続きの世界** として見せ、絵本を手に取る人にも、遊びから入る子にも、自然に馴染む見え方にします。広告的な強調 (赤ボタン・煽り文言・カウントダウン・「無料」断言・受賞や数値の誇示) は行わず、絵本の余白とやさしさをそのまま LP の余白に持ち込みます。

## 含まれるコンポーネント

### トークン (プレビュー)
- `tokens/colors.html` — カラートークン一覧プレビュー
- `tokens/typography.html` — タイポトークン (見出し / 本文 / キャプション) 一覧プレビュー
- `tokens/spacing.html` — スペーシングトークン (角丸 / 余白 / 影) 一覧プレビュー

### コンポーネント
- `components/hero/current.html` — 現状ヒーロー (改善前のベースライン。差分比較用に必ず残す)
- `components/hero/brief.md` — ヒーロー変種生成ブリーフ。要件と 3 案の方向性を定義
- `components/book-bridge/brief.md` — 「えほん×あそびば」並置セクション生成ブリーフ。絵本カードと遊び場カードを左右並置する 3 パターンの方向性を定義

### ヒーロー 3 案の方向性 (詳細は `components/hero/brief.md`)
- **A. 絵本ジャケット主役**: 左 60% に絵本ジャケット、右 40% にあそびば導線。絵本との出会いを重視。
- **B. 等価並置**: 絵本カードとあそびばカードを 50:50 で並べる。回遊重視。
- **C. 朝の森イラスト主役**: 全幅イラスト + 下部にミニ CTA 2 枚。世界観重視。

### 「えほん×あそびば」3 案の方向性 (詳細は `components/book-bridge/brief.md`)
- **A. 横並びカード**: 左=絵本 / 右=あそびば、同サイズ・同余白
- **B. 縦積みストーリー**: 上=絵本のページ抜粋 → 下=あそびば導線、物語的接続
- **C. 重ね合わせ**: 絵本ジャケットの余白にあそびばイラストを差し込むレイヤー構成

## ワークフロー

### 1. claude.ai/design 上での作業 (morito)

1. このプロジェクトを Claude Design で開く
2. `tokens/*.html` に目を通し、配色 / タイポ / 余白の基準を確認
3. `components/hero/brief.md` を読み、**ヒーロー 3 案を並列生成**
4. `components/book-bridge/brief.md` を読み、**「えほん×あそびば」3 案を並列生成**
5. それぞれ採用案を 1 つずつ選定し、HTML/CSS をエクスポート

### 2. Claude Code 側での反映

1. 採用案を Claude Code に渡し、本リポジトリの `index.html` の該当セクションに書き戻し
2. `develop-app` ブランチで作業 → staging (`pono-asobiba-app-staging.ndw.workers.dev`) で検証
3. 実機 (iPad / iPhone / PC) で表示・タップ・スクロール確認 → 問題なければ本番反映

## 注意

- **対象は LP (`index.html`) のみ**。`play.html` 以降のゲーム本体、`bento/`、`maze/`、`quizland/` などのゲームディレクトリは本 Brand Kit のスコープ外です
- ヒーロー以外のヘッダー/フッター/お問い合わせ等のセクションも今回は触りません
- 装飾は絵本トーン優先。広告的な強調表現は採用しません
- NG 表現: 「今だけ」「限定」「残り◯名」等の煽り文言、`#FF0000` 系の純赤 CTA、500ms 未満の点滅、自動再生音、ポップアップモーダル
- 採用案エクスポート粒度: **セクション単位の完全な HTML + scoped CSS** (CSS 変数は `tokens/colors.html` の命名 `--bg-milk` / `--primary` / `--text-brown` をそのまま流用)。Claude Code 側には HTML ファイルをそのまま添付する (口頭サマリ NG)
- 既存の `current.html` は **絶対に消さない** こと。バリアント比較とロールバックの基準になります
