# Hero scattered labels 差し替え 生成ブリーフ

> このファイルは Claude Design (Pono LP Brand Kit) 上で 3 案を並列生成するための仕様書。生成対象は **LP (index.html) ヒーローセクションに散らばっている 5 つのクラフト紙シールラベルのうち、 ゲーム未実装で誇大表示となっている 3 種 (なぞなぞ / ぼうけん / いろあそび) の PNG** のみ。 既存 ongaku / puzzle の質感を踏襲しつつ、 実装ゲームと整合する案 C ハイブリッド (ジャンル 3 + 育つ力 2) への着地を確定済。 morito が 3 案から採用案を選び、 Claude Code が assets/lp/hero/labels/ に保存 → index.html セレクタ rename → sw.js バンプ → push の流れ。

---

## 目的

- **実装ゲームとの整合性確保**: 現状の 「もじ」 「おえかき」 ラベルはゲーム未実装で誇大表示。 実装済の quizland (なぞなぞ) / maze (ぼうけん) / bento (いろあそび) に差し替え、 LP の正直さを担保
- **既存クラフト紙質感の踏襲**: ongaku.png / puzzle.png のベージュ・クラフト紙手書き調を完全に揃え、 5 シール並びでビジュアル分断を起こさない
- **「ジャンル 3 + 育つ力 2」 のハイブリッド**: 案 C 確定 (なぞなぞ + ぼうけん + いろあそび + おんがく + パズル の 5 枚並び)
- **5 シール並びでの視覚バランス**: bbox 高さ・モチーフ配置を揃え、 散らし配置時に一枚だけ浮かない統一感

## 共通要件

### 既存スタイル踏襲 (必読・リファレンス)

採用後に並ぶ既存 2 枚 (ongaku.png, puzzle.png) の特徴を完全に揃えること。

- **ongaku.png** (241×146 1x, クラフトカード型):
  - 形状: 角丸長方形のクラフトカード
  - 色: ベージュ・クラフト紙地 (`#E8D9BD` 〜 `#D9C4A0` 系の紙色)
  - 文字: 「おんがく」 焦茶手書き風 (Zen Maru Gothic 900 風), `#5D4E37`
  - モチーフ: 右上に手書き 8 分音符 (茶, 文字より小さい)
  - 影: ほぼ無 (紙のごく淡いセルフシャドウのみ)

- **puzzle.png** (229×112 1x, パズルピース型):
  - 形状: パズルピース型 (凹凸あり、 やや薄緑紙質)
  - 色: 薄緑シック和紙 (`#D5DCC0` 〜 `#C0CCAA` 系)
  - 文字: 「パズル」 焦茶手書き風
  - モチーフ: 右下にオレンジ＋ティールのパズルピース 2 個 (小)
  - 影: ほぼ無

- **共通仕様**:
  - 透明背景 PNG
  - 焦茶 (`#5D4E37` 系) 手書き風文字
  - Zen Maru Gothic / Hiragino Maru Gothic Pro 風の太字 (weight 900 相当)
  - クラフト紙質感 (紙繊維のごく淡いテクスチャ可)

### 寸法ガイド

- **1x**: 220-240 px 幅 × 92-118 px 高 (既存 ongaku 241×146、 puzzle 229×112 の範囲内)
- **@2x**: 1x の **正確に 2 倍** (Retina 表示用)
- **bbox 高さ揃え必須**: 5 シール並びでの統一感のため、 既存 ongaku/puzzle と高さの "見え方" を揃える (1x 高 100±10 px 目安)
- **文字字数差の吸収**: 「いろあそび」 は 5 字で他より長いため、 文字サイズを微調整して bbox 横長で対応 (後述)

### typography

- 各シール **1 行ひらがな** (3-5 字)
- 色: 焦茶 `#5D4E37`
- ウェイト: 太字 (weight 900 相当)
- フォント: Zen Maru Gothic / Hiragino Maru Gothic Pro 風の手書き調
- 文字単体は変形・斜体・極端な装飾フォント禁止

### モチーフ配置ルール

- 文字を邪魔しない端 (右上 or 右下 or 左下) に **小モチーフ 1 個** を配置
- 既存 ongaku は右上音符、 puzzle は右下ピース。 新規 3 種も同等の配置ロジックを踏襲
- モチーフは文字より明らかに小さく (文字高の 40-60%)
- モチーフ色は焦茶ベース + ワンポイントカラーまで許容 (案 C で色彩アクセント強化)

### アクセシビリティ

- 装飾扱い (`alt=""` + `aria-hidden="true"`) で SR 非読み上げ。 PNG 内のテキストは alt に書かない
- `prefers-reduced-motion` は CSS 側で停止済 (PNG は静止画なので影響なし)
- 文字コントラスト: 焦茶 on ベージュは AA 余裕クリア (3 案とも維持)

## 3 案の方向性 (Claude Design 並列生成用)

### 案 A — クラフト紙 baked 系 (既存踏襲 / 推奨)

- **キャラクター**: 既存 ongaku/puzzle と完全同質感。 クラフト紙地 + 焦茶手書き文字 + 小モチーフ
- **質感**: 紙繊維テクスチャ薄め、 印刷の擦れ感を再現
- **配置**: モチーフは既存 2 枚と同様、 右上 or 右下のみ
- **狙い**: 5 シール並びの完全統一感。 採用後の置換違和感ゼロ
- **避ける**: 過度な色彩アクセント、 既存 ongaku/puzzle と質感ズレ

### 案 B — シール工作系 (ちぎり紙風)

- **キャラクター**: もう少し手作り感を強める。 ちぎり紙風の縁の不規則さ、 ハサミで切った跡風
- **質感**: 紙の縁が rectangular でなく微妙にギザギザ or 波打ち
- **配置**: モチーフは案 A と同じ右上/右下基本だが、 紙の不規則さを活かして左下配置も許容
- **狙い**: 「子供が貼ったシール」 感の演出、 親しみやすさ強化
- **避ける**: 縁の不規則さを強調しすぎてプロダクト感を損なう、 ongaku/puzzle と並べた時の浮き

### 案 C — 色彩アクセント系

- **キャラクター**: 基本はベージュ・クラフト紙だが、 文字色 or モチーフに 1-2 色の色彩アクセントを強める
- **質感**: ベース紙質は維持 (案 A と同等) しつつ、 アクセントカラーで子供の視線を引く
- **配置**: モチーフは右上/右下基本、 アクセントカラーは Pono ブランドの `--primary: #F2915A` (オレンジ) or `--primary-light: #FDDCBF` を流用
- **狙い**: 子供目線の楽しさ強化、 LP 全体の視覚的ポップさ向上
- **避ける**: 蛍光・ネオン系、 既存ベージュ系統からの逸脱

> **採用後の置換ターゲットは 3 種 (nazonazo / bouken / iroasobi)**。 各案で 3 種をセットで生成し、 morito が案単位で採用 (案 A 全採用 or 案 C 全採用) する。 シール単位での mix-and-match は質感ズレを生むため非推奨。

## 個別シール画像仕様 (3 種詳細)

### 1. nazonazo (quizland 対応, ジャンル)

- **文字**: 「なぞなぞ」 (4 字、 ひらがな)
- **モチーフ案**:
  - 第一候補: 手書き風 「**？**」 マーク (焦茶、 文字より小さく)
  - 第二候補: ふくろうシルエット (quizland マスコット相当、 簡略化)
- **推奨配置**: 右上 (ongaku と並列感)
- **bbox**: 220-235 px × 100-115 px (1x), 4 字標準
- **注意**: 「？」 マークは小さく抑える (主役は文字)

### 2. bouken (maze 対応, 育つ力)

- **文字**: 「ぼうけん」 (4 字、 ひらがな)
- **モチーフ案**:
  - 第一候補: 手書き風ランタン (温かい琥珀色の灯り、 maze のキーアイテム)
  - 第二候補: 足跡 (小さな足跡 2-3 個)
  - 第三候補: 葉っぱシルエット (森のたんけん感)
- **推奨配置**: 右上 or 左下
- **bbox**: 220-235 px × 100-115 px (1x), 4 字標準
- **トーン**: 「やさしい」 印象を保つため、 育つ力ラベルは文字のウェイトを案 A 標準 (weight 900) より気持ち抑えても良い

### 3. iroasobi (bento 対応, 育つ力)

- **文字**: 「いろあそび」 (5 字、 ひらがな、 **他より長い**)
- **モチーフ案**:
  - 第一候補: クレヨン 3 本 (赤・黄・緑、 案 C のアクセントカラー候補)
  - 第二候補: 色パレット片 (絵の具のドット 3 色)
- **推奨配置**: 右上 or 左下
- **bbox**: 230-245 px × 100-115 px (1x), **横長気味** で 5 字を吸収
- **注意**:
  - 5 字なので文字サイズをやや小さく (4 字シールの 85-90%) して bbox 横長で対応
  - bouken と同じく 「やさしい」 印象を保つ

## 出力形式

- **透明背景 PNG** (アルファチャンネル必須)
- **1x + Retina @2x の 2 枚 1 セット**
- **合計 6 PNG** (3 シール × 2 解像度) を Claude Design からエクスポート
- **ファイル名 (確定)**:
  - `nazonazo.png` / `nazonazo@2x.png`
  - `bouken.png` / `bouken@2x.png`
  - `iroasobi.png` / `iroasobi@2x.png`
- **配置先**: `d:/AppDevelopment/pono-asobiba-app/assets/lp/hero/labels/`
- **ファイル名のリネーム禁止**: 採用後の index.html セレクタは上記ファイル名前提

## NG (3 案共通・絶対やらない)

- ベージュ系統から外れる **蛍光色 / ネオン**
- 文字が読みにくい **斜体・極端な装飾フォント**
- 「？」 が露骨に大きい (主役は文字、 モチーフは脇役)
- 「もじ」 「おえかき」 「かず」 等の **旧表現を再利用**
- 既存 ongaku の音符 / puzzle のパズルピースと **同じモチーフ** (重複回避)
- **影が強すぎる** (既存はほぼ無影、 紙のごく淡いセルフシャドウのみ許容)
- **キャラクター描画** (ポノ / ハリネズミ / その他キャラの顔・全身) → ラベルは文字主体、 キャラはヒーロー本体側に存在
- **JPEG 出力** (透明背景必須のため PNG のみ)
- **背景色付き** (透明背景必須、 紙質テクスチャは紙地そのものを描く)
- **過度な紙焼け・シミ** (kids 向けなのでクリーン寄り)

## 採用後のフロー

1. Claude Design で 3 案 (A/B/C) を並列生成、 各案で 3 種 (nazonazo / bouken / iroasobi) × 2 解像度 = 6 PNG ずつ
2. morito が採用案を選定 (案単位、 シール mix-and-match 非推奨)
3. 採用案の 6 PNG を Claude Code に直接コピペ or file 渡し (mojibake 回避のため 1 枚ずつチャットも可)
4. Claude Code が `assets/lp/hero/labels/` に保存 + `index.html` セレクタ rename (旧 moji/kazu/oekaki → 新 nazonazo/bouken/iroasobi)
5. `sw.js` の `CACHE_VERSION` をバンプ (キャッシュ対象 PNG + index.html 編集、 SW update 検知のため +2 推奨)
6. `develop-app` ブランチで push → App staging (`https://pono-asobiba-app-staging.ndw.workers.dev/`) で実機確認
7. iPhone / iPad / PC の 3 BP で散らし配置の視覚バランスを確認
8. OK なら本番ブランチへマージ (master へ反映)

## メモ (morito 向け)

- **推奨は案 A** (既存 ongaku/puzzle と最も揃う)。 案 B/C は発展余地として並列確認用
- 案 C 色彩アクセント系は、 子供の視線誘導力は強いが、 既存 ongaku/puzzle との質感ズレリスクあり → 採用時は既存 2 枚も同質感に再生成する 「全 5 枚刷新」 を視野に入れること
- **puzzle.png のみ薄緑紙質** で他 4 枚と質感が違う既存問題は別タスク (将来 puzzle.png 再生成も検討余地)
- 旧 PNG (moji.png / kazu.png / oekaki.png + @2x) はリポジトリ内に履歴として残す or v1484 revert 同様に削除のどちらでも可 (採用後判断)
- **bouken / iroasobi は「育つ力」** 用語なので、 文字フォントの太さを案 A 標準より少し抑えて 「やさしい」 印象にしても良い (ジャンル 3 種 = なぞなぞ/おんがく/パズル と差別化)
- 過去事故 (v1484 で nazonazo/bouken/iroasobi を実装したが質感ズレで revert 06f564e / sw v1485) を踏まえ、 今回は **brief 段階で既存質感を完全コミット** してから生成に入る
