# 「ちかぢか とどくよ」セクション 生成ブリーフ

> このファイルは Claude Design 上で 3 案を並列生成するための仕様書。生成対象は **LP 中段に挿入する「アプリ化予告 (Coming Soon)」スリムバナーのみ**。LP は morito 戦略再解釈により「遊び場コミュニティ醸成 LP」へ転換済みで、 3〜6 ヶ月以内のアプリ化 (PWA / ネイティブ) を控えている。 旧 coming-soon-a.html (Stitch v2) は phone モック + chip 群 + 長文 note で「情報量過多」「宣伝臭」「うざい」と判定されたため、 本ブリーフでは **「細い帯 + アプリアイコン + クリックで詳細モーダル」** に方針を一新する。

---

## 目的

- 親に **「あ、アプリ版がちかぢか来るんだ」** とそっと伝える (押し付けない、 期待醸成のみ)
- 子供には **アプリアイコンのポノの顔** で「これ、ホーム画面に置きたい」と感じてもらう
- LP 本体の流れ (ヒーロー → あそびばで できること → Amazon バナー) を **邪魔しない 1 行帯** に収め、 詳細はクリック時のモーダルに閉じ込める
- 3〜6 ヶ月後のアプリ化施策への **ソフトランディング** (memory: project_lp_strategy_shift)

---

## セクション全体の構造

- **配置**: 「あそびばで できること」セクションの直下、 Amazon バナー帯の **上** (LP 縦動線の中段)
- **形態**: 横長スリムバナー (案により高さ 100〜180px)。 セクション見出しは **なし** (帯そのものが UI)
- **構成要素 (左→右、 共通)**:
  1. **左**: 短いコピー 1 行 + (任意でサブ 1 行)
  2. **右**: 角丸正方形のアプリアイコン (ポノの顔ドアップ)
  3. **右端**: → 矢印 or 「タップで くわしく」テキスト (クリッカブル示唆)
- **挙動**: バナー全体が `<button>` または `<a>` で 1 クリック可能。 クリックで詳細モーダルを開く
- **背景**: `--bg-milk` ベース + 薄い `--primary-light` 帯 or ノート罫線風の控えめな装飾

### 残すコピー (3 案共通の唯一の必須コピー)

> **「ポノの あそびばが アプリに なります。 ゲームも コンテンツも うんと ボリュームアップ。」**

- 案 A はこれを 1 行に圧縮、 案 B は 2 行に折る、 案 C はメイン 1 行 + サブ 1 行に分けるなど、 配置の自由度はあり
- 「いつ公開」は **断言しない**。 「ちかぢか」「もうすぐ」「じゅんびちゅう」のいずれかでぼかす

### 削除するもの (旧 coming-soon-a.html から廃止)

- phone モック内の grid 詳細描画
- chip 4-6 個の機能羅列 (「もじ」「かず」「リズム」等)
- app-note 長文補足
- notify 長文 (メール登録系の文言含む)

## 共通要件

### viewport / breakpoints
- **390px (iPhone) / 768px (iPad縦) / 1200px (PC)** の 3 BP すべてでプレビューを生成
- viewport meta は `width=device-width, initial-scale=1.0` のみ。 `user-scalable=no` は使わない

### typography
- 本文: `Zen Maru Gothic` weight 700、 fallback `Hiragino Maru Gothic Pro`
- 強調語 (「アプリに なります」「ボリュームアップ」等): weight 900
- サイズは `clamp(14px, 3.4vw, 20px)` 程度。 スリム帯のためフォントサイズ控えめ
- ふりがなが必要な漢字 (例: 「届」「予告」等) は使わない方針。 全ひらがな + カタカナで完結させる

### colors (tokens/colors.html 準拠)
- `--bg: #FFF8F0` (ミルク色背景)
- `--primary: #F2915A` (オレンジ)
- `--primary-deep: #C96A2E` (濃いオレンジ、 アイコン縁取り)
- `--primary-light: #FDDCBF` (薄オレンジ、 帯背景)
- `--text: #5D4E37` (ブラウン)
- `--text-light: #A89880`
- `--yellow: #FFE07A` (案 B のマーカー風アクセント)
- バナー外枠の影: `0 2px 8px rgba(93, 78, 55, 0.08)` のごく淡いもの

### アクセシビリティ
- **WCAG AA** コントラスト 4.5:1 以上 (大型 3:1)
- バナー全体が `<button type="button">` または `<a href="#coming-soon-modal">` で `:focus-visible` 時に 2px outline (`--primary`) 必須
- `aria-label="ポノのあそびば アプリ化のおしらせを ひらく"` 等、 クリック先が詳細モーダルである旨を明示
- `prefers-reduced-motion: reduce` 時、 hover/load アニメーション全停止
- min タップターゲット 48px (バナー高さ 100px 案でも内部のクリック領域は 100px 確保)
- アイコン `<img>` の `alt="ポノのアプリアイコン"` (装飾でなく意味画像)

### コピー制約
- **「無料」断言禁止** → アプリの料金には一切触れない
- 「いつ公開」断言禁止 → 「ちかぢか」「もうすぐ」「じゅんびちゅう」のいずれか
- 「今だけ」「限定」「残り◯名」「先着」等の煽り NG
- 受賞バッジ・利用者数の架空数字 NG (景表法)
- 「Coming Soon」英語表記は装飾としてのみ可。 主コピーは日本語ひらがな主体

### スコープ規律
- `:root` 再宣言禁止 → 必ず `.coming-soon` (or section スコープ) に CSS 変数を退避
- staging 絶対 URL 禁止 → `assets/images/...` 相対パス
- inline `onerror` の JS 避ける → CSS only fallback
- 既存ヒーロー / あそびばで できること / Amazon バナーの CSS 変数を上書きしない
- モーダル開閉 JS は inline ではなく、 `<script>` ブロックでまとめる (採用後 Claude Code が `index.html` に組み込む際に分離しやすい形)

## 3 案の方向性

### 案 A: 横長 1 行帯型 (Minimal Banner)

最も控えめ、 LP の流れを一切邪魔しない 1 行帯。

- **構成 (左→右)**:
  1. 左 70%: コピー 1 行「**ポノの あそびばが アプリに なります。 ゲームも コンテンツも うんと ボリュームアップ。**」(clamp(14px, 3.2vw, 18px))
  2. 右 25%: アプリアイコン 56〜64px 正方形 (角丸 14px、 オレンジ縁取り、 ポノ顔)
  3. 右端 5%: `→` 矢印 (`--primary-deep`、 hover で +4px translateX)
- **高さ**: max-height 100〜120px (BP 共通、 内側 padding 12-16px)
- **背景**: `--bg` 単色 + 下罫線 1px (`--primary-light`)
- **モバイル時**: 同レイアウトを縮小、 コピーが 2 行に折れても OK
- **狙い**: 「お知らせバー」感。 親が「あ、 そうなんだ」と一瞬で認識して読み流せる軽さ。 押し付けゼロ
- **避ける**: 装飾過多、 phone モック、 chip 群、 「Coming Soon」大文字英語の主役化

### 案 B: リボン/タグ型 (Sticker Banner)

絵本工作感のあるシール風バナー。 帯としては薄いが、 ちょっと目を引く。

- **構成**:
  1. 全体を -1.5deg ほど軽く回転、 dashed border 2px (`--primary-light`)
  2. 左側にコピー、 黄色マーカー (`--yellow`、 半透明 60%) を強調語「**アプリに なります**」の背景に下敷き
  3. 右側にアプリアイコン 64〜72px、 バナーから少しはみ出すように `translateY(-8px)` で配置 (シール感)
  4. 右下に小さく「タップで くわしく」(11-12px、 `--text-light`)
- **高さ**: max-height 130〜150px
- **背景**: `--bg` + うっすらノート罫線風 grid (CSS gradient、 opacity 0.08)
- **モバイル時**: 回転は -1deg に弱める、 アイコンはみ出しは維持
- **狙い**: 子供 (と子供心のある親) が「シール貼ってある！」と感じる工作タッチ。 絵本 LP との地続き感
- **避ける**: 派手な回転 (>3deg) で読みづらくなること、 マーカーが赤系で煽り感を出すこと

### 案 C: カード予告型 (App Preview Card)

少し背の高い角丸カード。 サブテキスト 1 行で内容を匂わせる、 book-aside / bookcheer と同類の高さ感。

- **構成 (PC: 左右 2 カラム / モバイル: 縦積み)**:
  1. **左 60%**: メインコピー (1 行 or 2 行折り返し) + サブ 1 行「**ゲーム、 おはなし、 まいにちの あそび。 もっと たっぷり おとどけ。**」(13-14px、 `--text-light`)
  2. **右 40%**: アプリアイコン 80〜96px (角丸 18px、 オレンジ縁取り、 軽い drop shadow)
  3. カード右下に控えめな CTA テキスト「**くわしく みる →**」(`--primary`、 14px)
- **高さ**: max-height 180px (PC) / 200px (モバイル縦積み時)
- **背景**: `--bg-milk` + `--primary-light` から透明への薄いグラデーション (左→右)
- **モバイル時**: 縦積み (コピー上、 アイコン下中央、 CTA テキスト下)
- **狙い**: 「ちゃんと予告カードがある」安心感。 親が「これ、 アプリの紹介セクションだな」と認識しやすい。 アプリ化後、 そのまま「ダウンロードへ」カードに転用しやすい構造
- **避ける**: chip 群の復活、 phone モック復活、 高さ 220px 超え (本セクションが LP の主役になってしまう)

## 詳細モーダル仕様 (3 案共通)

バナークリックで開くモーダルの仕様。 既存の `.game-modal` (GAME_MODAL_DATA 系) パターンを踏襲推奨。

### 構造

- **オーバーレイ**: `rgba(93, 78, 55, 0.45)` の半透明ブラウン、 `backdrop-filter: blur(4px)` (対応ブラウザのみ)
- **モーダル本体**: 角丸 24px、 `--bg` 背景、 max-width 480px (PC) / 92vw (モバイル)、 上下マージン中央寄せ
- **ヘッダー**: 「**いま つくっています**」セクションタイトル (Zen Maru 900、 clamp(20px, 4.5vw, 26px))
- **アイコン拡大表示**: バナーと同じポノアプリアイコンを 120〜140px で中央配置 (オレンジ縁取り維持)
- **チラ見せリスト** (箇条 3〜4 個):
  - 「もっと たくさんの ゲーム」
  - 「あたらしい あそびば」
  - 「ポノからの とくべつな おしらせ」
  - 「(任意) ホーム画面に おける アプリアイコン」
- **フッターテキスト**: 「**じゅんびが できたら、 ここで おしらせするね。**」 (Zen Maru 700、 14-15px、 `--text-light`、 中央寄せ)
- **閉じるボタン**: 右上 × ボタン (48px タップ領域、 `aria-label="とじる"`)

### 挙動

- 開閉トリガー: バナーのクリック / Enter / Space
- 閉じる方法: × ボタン / オーバーレイクリック / Escape キー の 3 系統すべて対応
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby="coming-soon-modal-title"`
- 開いた瞬間にフォーカスを × ボタンへ移動、 閉じた後はバナーへ戻す (focus trap 推奨)
- `prefers-reduced-motion: reduce` 時、 fade-in/out は 0.01s に短縮 (実質瞬時)
- **自動ポップアップではない** (ユーザーのクリックでのみ開く → README NG リスト「ポップアップモーダル」抵触なし)

## 使う画像 (アプリアイコン)

### 推奨 (default)

- **`assets/images/characters/pono/pono_face_circle.png`** ⭐
  - 顔ドアップ + 円形クリーム背景が既に baked 済み。 正方形角丸に `object-fit: cover` で即決まる
  - スリムバナー右端の小さい矩形 (40〜96px 想定) でも顔が潰れず視認性 max
  - LP ヒーローの bear+wordmark baked ロゴと「同じ baked 思想」で統一感

### 代替候補 (2 つ)

- **`assets/images/characters/pono/dance_hooray.png`**
  - 「手を上げてる」要望に最適合。 顔 + 両手バンザイの bust-up クロップで使用可
  - hover/tap 時の差し替え二段構成にも転用可
- **`assets/images/characters/pono/dance_hi.png`**
  - dance_hooray の代替。 腕が水平寄りで横長アイコンにも収まりが良い

### アイコン枠 CSS 仕様 (推奨実装)

- **サイズ**: 案 A 56-64px / 案 B 64-72px / 案 C 80-96px (Retina 用に 2x asset)
- **角丸**: `border-radius: 14px` 〜 `18px` (iOS app icon 比率 ≒ 25%)
- **縁取り**: `box-shadow: 0 0 0 2px var(--primary), 0 2px 6px rgba(0,0,0,.12)`
- **背景**: 透過のまま (画像 baked クリーム色をそのまま見せる)
- **顔配置**:

```css
.app-icon img {
  width: 100%;
  height: 100%;
  object-fit: cover;
  object-position: center 38%;  /* 鼻〜口を中央へ */
  transform: scale(1.05);       /* 角丸 trim 縁透け防止 */
}
```

## NG (3 案共通・絶対やらない)

- ❌ **自動ポップアップモーダル** (クリック発火のみ。 README NG リスト準拠)
- ❌ **自動再生音声** (バナー出現音、 モーダル開閉 SE すべて禁止)
- ❌ **500ms 未満の点滅・派手な bounce / 連続 spin**
- ❌ **「今だけ」「限定」「残り◯名」「先着」等の煽り文言** (景表法 + ブランド毀損)
- ❌ **「無料」「完全無料」断言** (3 ティア構成のため、 アプリ料金にも触れない)
- ❌ **公開日断言** (「2026 年夏」「○月リリース」等)
- ❌ **`#FF0000` 系の純赤 CTA** (`--primary` オレンジ系で統一)
- ❌ **受賞バッジ / 「No.1」表記 / 利用者数の架空数字**
- ❌ **phone モック / chip 群 / app-note 長文 / notify 長文の復活** (旧 coming-soon-a.html の失敗パターン)
- ❌ **メール登録フォーム** (本ブリーフでは対象外。 将来別セクションで検討)
- ❌ **ポノ / ハリネズミ / その他キャラクターの新規再描画** (既存絵柄を流用)

## 採用後のフロー

1. Claude Design 上で 3 案 (A / B / C) を 3 BP × 3 案 = 9 プレビューで並列生成
2. morito が 3 案を見比べ、 採用案 1 つを選定 (or 細部 mix-and-match 指示)
3. Claude Code 側で `DesignSync` (or `/design-sync pull`) を実行し採用案を取り込み
4. Claude Code が `d:/AppDevelopment/pono-asobiba-app/index.html` の **「あそびばで できること」直下 / Amazon バナー上** にバナー + モーダルを挿入 (既存 CSS 変数・既存アセットパスとの整合性チェックを含む)
5. `js/sw.js` の `CACHE_VERSION` をバンプ (sw.js キャッシュ対象を編集したため必須)
6. `develop-app` ブランチで staging (`https://pono-asobiba-app-staging.ndw.workers.dev/`) に反映 → 実機 (iPhone/iPad/PC) で 3 BP + モーダル開閉 a11y 確認
7. OK なら本番ブランチ (`master`) へマージ

## メモ (morito 向け)

- **本命は案 A (1 行帯)** と想定。 LP 全体の縦の流れを最も邪魔せず、 「お知らせバー」として自然に読み流せる
- **案 B (リボン)** は遊び心狙いだが、 LP 中段に置くとヒーローの v1358 baked ロゴとの「工作タッチ被り」が起きる可能性あり → 採用するなら回転角を弱め (-1deg) 推奨
- **案 C (カード)** はアプリ化後に「ダウンロードへ」カードに転用しやすいので、 中長期的な再利用性で選ぶ価値あり。 ただし高さ 180px は LP 中段ではやや主張強め
- ハイブリッド案: **案 A レイアウト + 案 B の黄色マーカー強調語** だけ採用、 という mix も視野
- 旧 coming-soon-a.html (Stitch v2) は **削除せず保持** (README ルール「current.html は絶対消さない」 + バリアント比較の基準として残す)
- アプリアイコン差し替え (`pono_face_circle.png` → `dance_hooray.png` の hover 切替) はバナー採用後の Phase 2 で検討
