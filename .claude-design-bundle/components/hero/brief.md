# Pono LP ヒーロー — Claude Design バリアント生成ブリーフ

> このファイルは Claude Design 上で 3 案を並列生成するための仕様書。生成対象は **「ポノのあそびば」LP (index.html) のヒーローセクションのみ**。LP は こだまの森出版の絵本『ありがとうって、うれしいね 〜くまのこポノのちいさなものがたり〜』販売 LP であり、あそびばは世界観体験の入口。ファネルは「あそぶ → 面白い → 絵本も読みたい」。

---

## 共通要件

### ターゲット / トーン
- **対象**: 3〜6 歳の子ども + 保護者 (読み聞かせ・横で見守る親世代 30〜40 代)
- **語り手**: 「ネモ先生」名義。低圧・誘導しない・余白を残す丁寧語
- **brand_voice**: やさしい / あたたかい / てづくり感。絵本の地続き感を出す。商業広告色は出さない
- **コピー指針**: ひらがな多めの短文 (「あそびば」「いりぐち」「みてね！」)。clamp() で可変、改行は `<br class="sm-only">` 等で BP 毎制御

### ビジュアル / デザインシステム
- **colors**:
  - `--bg: #FFF8F0` (ミルク色背景)
  - `--primary: #F2915A` (くま/木のオレンジ)
  - `--primary-light: #FDDCBF`
  - `--text: #5D4E37` (ブラウン)
  - `--text-light: #A89880`
- **typography**: `Zen Maru Gothic` weight 700 (本文) / 900 (見出し)。fallback: `Hiragino Maru Gothic Pro`
- **radius**: 14〜28px (角丸はやわらかく)
- **shadow**: `0 4px 14px rgba(93, 78, 55, 0.12)` 系の淡い影のみ。ドロップ強調 NG
- **絵文字**: 🌳🌱🎵🐻🦔 等を 1〜2 個まで装飾として許可
- **アニメーション**: 0.4〜0.8s のふわっと出現のみ。bounce や派手な spin は避ける

### レスポンシブ
- **breakpoints**: 390px (iPhone) / 768px (iPad縦) / 1200px (PC)
- **3 BP すべてのプレビューを生成すること**
- viewport meta は `width=device-width, initial-scale=1.0` のみ。`user-scalable=no` は **使わない** (LP は a11y 重視、ピンチズーム可能を保証)

### アクセシビリティ
- **WCAG AA コントラスト** 必須 (テキスト 4.5:1、大型 3:1)
- 装飾画像は `alt=""`、意味画像は日本語 alt
- CTA ボタンは min-height 48px (タップターゲット)
- ロゴ・キャラクター画像は `loading="eager"`、それ以外は `loading="lazy"`

### コピー制約 (重要)
- **「無料」を断言しない**。実際には free / book特典 / sub の 3 ティア構成。許容表現:
  - 「**きほんの あそびは むりょう**」
  - 「すぐにあそべるよ」
  - 「いりぐちは ひらいてるよ」
- **Amazon バナーは削除しない** (絵本販売が LP の主目的)。ただし配置の優先度は案ごとに変えてよい

---

## 案 A — 世界観優先 (World-First)

絵本の世界に直接連れて行く案。**「絵本の延長としてのあそびば」を最優先**。

- **構成 (上→下)**:
  1. 絵本見開き風の **横長ビジュアル** (既存の `assets/images/A+用ヘッダー.jpg` を活用、トリミングや淡いマスク許可)
  2. 中央に手書き風コピー「**えほんの ポノと あそぼう**」(Zen Maru 900, clamp(28px, 6vw, 56px))
  3. サブコピー「ネモ先生がつくった ちいさな あそびばだよ 🌳」
  4. CTA: **「あそびばの いりぐち →」** (#F2915A、角丸 24px、影あり、min-height 56px)
  5. その下に小さく「きほんの あそびは むりょう」と添える
  6. **Amazon バナーは中段に降格** (ヒーローの直下ではなくスクロール一画面分下、見出し「えほんも あるよ 📖」付き)
- **狙い**: 親に「これは広告じゃなく絵本の世界の続き」と先に伝える。CTR より滞在感
- **避ける**: 体験スクショの羅列、機能チップ、対象年齢の数字明示

## 案 B — 体験予告優先 (Play-First)

「中で何ができるか」を上で見せる案。**遊びの具体性で 3〜6 歳の親に刺す**。

- **構成 (上→下)**:
  1. ヒーロー内に **「あそびば プレビュー」枠** (3〜4 個のミニサムネを横スクロールまたはモザイク配置、CSS で擬似 GIF 風にゆっくり切替も可)
     - 例: ひらがななぞり / おえかき / めいろ / おべんとう
  2. 見出し「**みて、さわって、あそぼう**」
  3. **チップ群** (角丸 pill, --primary-light 背景, --text 文字):
     - `2〜7さい` `もじ` `かず` `おえかき` `パズル`
  4. CTA: **「すぐ あそぶ →」** (--primary, 大型)
  5. サブ: 「きほんの あそびは むりょうだよ」
  6. Amazon バナーは CTA の直下にコンパクトに配置 (横長 1 行、見出し「もとになった えほん」)
- **狙い**: 流入直後の親に「何ができるか」を 3 秒で伝える。離脱抑制
- **避ける**: 「100 万人が遊んだ」等の架空数字、受賞バッジ、★レビュー風 UI

## 案 C — ハイブリッド (Bridge)

絵本と遊び場を **横に並べて橋渡し** する案。意思決定を保護者に委ねる。

- **構成 (PC: 2 カラム / モバイル: 縦積み)**:
  1. **左**: 絵本書影 (Amazon 商品画像 or 表紙アセット) + 「えほんを よむ」サブ CTA (枠線ボタン、--primary border)
  2. **右**: イラストカード (ポノとあそびばのアイコン) + 「**あそびばの いりぐち →**」メイン CTA (--primary 塗り)
  3. その下にブリッジテキスト (中央寄せ、clamp(16px, 3.6vw, 22px)):
     > **おなじ ポノが、えほんにも、あそびばにも。**
  4. さらに下に小さく「きほんの あそびは むりょう / えほんは Amazon で どうぞ 📖」
- **モバイル時**: 書影 → ブリッジ文 → あそびば CTA → Amazon CTA の順 (あそびば優先で離脱低減、ただし書影は最初に視界に入れる)
- **狙い**: 絵本販売と遊び場流入の両 KPI を取りに行く。保護者の選択肢提示型
- **避ける**: 2 つの CTA を同サイズにしてどちらも弱くすること。**メインはあそびば**、絵本はサブ強度で

## NG (3 案共通・絶対やらない)

- ❌ **受賞バッジ / 「No.1」表記 / 利用者数の架空数字** (景表法・優良誤認)
- ❌ **ポノ / ハリネズミ / その他キャラクターの新規再描画**。既存絵柄 (assets/images/, bento/assets/, 各ゲームの spritesheet) を流用すること
- ❌ **「完全無料」「ずっと無料」の断言** (3 ティア構成のため)
- ❌ **Amazon バナーの削除**
- ❌ **user-scalable=no / maximum-scale=1** (a11y 違反)
- ❌ **派手な bounce / 連続 spin / 自動再生音声**
- ❌ **テンプレ感のある SaaS ヒーロー** (グラデ背景に白文字大見出し + 紫ボタン、的なもの)

## 生成後のユーザー作業フロー

1. Claude Design 上で **案 A / B / C を 3 BP × 3 案 = 9 プレビュー** で並列生成
2. morito が 3 案を見比べ、採用案 1 つを選ぶ (or ハーフ採用で要素ミックス指示)
3. Claude Code 側で `/design-sync pull` を実行し、採用案を取り込み
4. Claude Code が `d:/AppDevelopment/pono-asobiba-app/index.html` の **ヒーローセクションのみ** を差し替え (Amazon バナー位置・既存 CSS 変数・既存アセットパスとの整合性チェックを含む)
5. 差し替え後、`sw.js` の `CACHE_VERSION` をバンプし、`develop-app` ブランチで staging (`pono-asobiba-app-staging.ndw.workers.dev`) に反映 → 実機 (iPhone/iPad/PC) で 3 BP 確認
6. OK なら本番ブランチへマージ
