# book 限定特典シール + 特別表紙 生成ブリーフ (tier v3)

> このファイルは Claude Design (Pono LP Brand Kit) 上で並列生成するための仕様書。生成対象は **絵本『ありがとうって、うれしいね』を買った人 (book tier) だけに贈られる特典シール 3 枚 (新規)** と、**シール帳の book_buyer_edition 特別表紙 (3 面)**。既存の余剰シール 5-6 枚 (下記参照) は選別で対応済のため再生成しない。採用後は `assets/ui/gacha/stickers/` へ配置 → `assets/data/game-stickers.json` に `book_bonus` タグで登録 → `Prototypes/StickerBookThreeJS/main.js` の `BOOK_VARIANTS` へ紐付け、の流れ。

## 0. コンテキスト

- 対象プロジェクト: `pono-asobiba-app` (3〜6 歳向け知育 Web アプリ)。絵本『ありがとうって、うれしいね』(こだまのもり出版) を買った人だけの特典を実装する tier v3 施策の一部

- 用途: パスワード (あいことば) 入力の welcome 演出でまとめて贈呈される「book 付録シール 8 枚」のうちの一部 + シール帳限定表紙。無料 (free) / サブスク (sub) ユーザーには表示されない、book 購入者だけの特別感を作る素材
- **既に選別済で再生成不要な既存シール** (`assets/ui/gacha/stickers/book_bonus_*_20260701.png`, 512×512 PNG):
  - `book_bonus_pono_book_rare` — 絵本表紙ミニチュア相当
  - `book_bonus_mama_pono_super` — ポノとお月さま系シーン
  - `book_bonus_hedgehog_rare` — 絵本の仲間キャラ単体
  - `book_bonus_friends_super` — 森の仲間たちシーン
  - `book_bonus_ehon_medal_super` — 記念メダル系装飾
  - これらの質感・色数・線の太さを **新規 3 枚のリファレンス** とすること (並べたときに浮かない統一感が最優先)
  - **注 (2026-07-06)**: `book_bonus_duck_rare`「アヒル」は 9→8 枚化に伴い catalog (`assets/data/game-stickers.json`) から削除済み。本ブリーフの発注 scope に変更はない (新規 3 枚は元々アヒルの代替ではない) が、リファレンス一覧からは除外した。

## 1. 世界観・トーン (全シール共通)

- **画風**: 絵本『ありがとうって、うれしいね』の水彩画タッチ。輪郭線はやわらかい焦茶の細線、塗りは水彩のにじみ・淡いグラデーション、ベタ塗りのフラットイラストは禁止
- **配色**: warm / orange / cream / wooden パレットを基調
  - 主人公ポノの毛色: 温かみのあるベージュ〜キャラメル系の茶
  - アクセント: `#F2915A` (オレンジ) / `#FDDCBF` (淡いオレンジ)
  - 背景・余白トーン (透過部にじみの縁取り等に使う場合): `#FFF8F0` / `#FFFDF8`
  - 線・文字: `#5D4E37` (焦茶)
- **雰囲気**: あたたかい・やさしい・静か。派手な効果音的表現 (爆発・スピード線・ネオン) は禁止
- **キャラクター描写**: ポノは絵本内の体型・顔立ち・毛並みと一致させること (別プロジェクトのアプリ内 3D/2D 立ち絵 `assets/images/characters/pono/` や `assets/images/bento/story/staff/pono_wave*.png` はタッチが異なるため **参照しない**。絵本原本のイラストトーンを最優先)

## 2. 新規制作シール 3 枚 (必須)

### シール 1: 「動くシール」用ベース 2 枚セット (ポノ静止 + 腕を上げたバリエーション)

- **用途**: CSS アニメーションで 2 枚を交互に差し替え、「ポノが手を振る」動きを表現する (JS のスプライトアニメではなく、2 静止画の CSS keyframe クロスフェード/切替が前提)
- **必要枚数**: 2 枚 1 セット
  1. `base` — ポノが正面向きで立って静止しているポーズ (両腕は自然に下ろす)
  2. `wave` — 同じ構図・同じ画角・同じ体の大きさで、**片腕だけを高く上げて手を振っている** ポーズ
- **絶対条件**: 2 枚の間で頭身・体のサイズ・立ち位置・カメラ距離・背景の有無を **完全に一致** させること (差し替えた瞬間に体がガタつく/浮く/縮むと違和感が出る)。変化していいのは腕の角度と表情 (笑顔の強さ) のみ
- **構図**: バストアップ〜全身、背景は透過 (単体シールとして貼れるシルエット)
- **表情**: にこやかな笑顔、目を細めた優しい表情

### シール 2: キャラクター集合「家族写真」風

- **用途**: 絵本に登場する主要キャラクター全員が並んで記念撮影しているような 1 枚絵
- **構図**: ポノを中心 (やや前列・中央寄り) に、絵本に登場する仲間の動物たち 5〜6 体を横並び〜緩やかな弧を描くように配置。全員がカメラ目線でにこやかにこちらを見ている「集合写真」のポーズ
- **登場キャラ候補**: ポノ (くまのこ) を中心に、既存 book_bonus シールに登場するキツネ・アヒル・ハリネズミ等、絵本内の仲間キャラから選定 (絵本原作に登場しないキャラの新規デザインはしない)
- **雰囲気**: 「はい、チーズ」的な微笑ましい記念撮影感。派手なポーズ・アクションシーンにはしない
- **背景**: 透過、または非常に淡い森の気配 (輪郭のみのぼかし) 程度に留め、キャラクターの視認性を優先

### シール 3: 「あいことば ありがとう」記念シール

- **用途**: パスワード (あいことば) 入力を完了した記念として贈られる、スタンプ・バッジ的な装飾シール
- **構図**: 円形または楕円形のスタンプ/バッジ風フレーム。中央に「あいことば ありがとう」の文字 (手書き風、ひらがな主体)、周囲を月桂樹風の葉飾り・小さな星・リボン等の祝祭モチーフで縁取る
- **文字**: `#5D4E37` の焦茶、丸みのある手書き風書体 (Zen Maru Gothic 900 相当の雰囲気)。感嘆符や記号の多用はしない
- **モチーフの色数**: 3 色以内 (焦茶 + オレンジ系アクセント 1-2 色) に抑え、既存 `book_bonus_ehon_medal_super` のトーンと揃える
- **NG**: 賞状・トロフィー等の「大人の表彰」感が強すぎる意匠、金ピカの過剰な装飾

## 3. 各シール共通の技術仕様

- **サイズ**: 1024×1024 px、正方形キャンバス内にシールモチーフを収める (モチーフ自体は正方形いっぱいに引き伸ばさない。余白を持たせた自然な配置)
- **フォーマット**: PNG、アルファチャンネル必須 (透明背景)
- **縦横比 stretch 禁止**: 実装時に `background-size:100% 100%` 等で歪ませることは絶対にしないため、**シール本体の比率は正方形キャンバスの中で自然に見えるよう設計者側で完結させる** こと (実装側で比率調整はしない前提)
- **線の太さ・塗りの密度**: 既存 `book_bonus_*` 6 枚と並べて浮かないこと (水彩のにじみ量・線の太さ・彩度を揃える)
- **ファイル名 (仮案、実装時に確定)**:
  - `book_bonus_wave_base_super_20260706.png` (シール1 base)
  - `book_bonus_wave_up_super_20260706.png` (シール1 wave)
  - `book_bonus_family_photo_super_20260706.png` (シール2)
  - `book_bonus_password_thanks_super_20260706.png` (シール3)
- **配置先**: `assets/ui/gacha/stickers/`

## 4. NG 要素 (全シール共通・絶対にやらない)

- URL / QR コード / SNS ロゴ・アイコンの描き込み
- 大人向けの語彙・漢字主体の文言 (ひらがな中心、対象年齢 3〜6 歳)
- 過度にレトロ・重厚・シリアスなデザイン (額縁調・古典絵画調・モノクロ写真調など)
- ベタ塗りのフラットイラスト・アニメ調の太いアウトライン (絵本の水彩タッチから逸脱)
- 蛍光色・ネオン・グラデーション多用
- ポノ以外のキャラクターを主役級に大きく描く (シール2 の集合写真を除き、ポノは常に主役)
- 絵本に登場しない新規キャラクターのデザイン追加
- JPEG 出力・背景色付き (透明背景 PNG のみ)
- 賞状・トロフィー等「大人の表彰式」感の強い意匠 (シール3)

## 6. 特別表紙 book_buyer_edition (シール帳、3 面)

- **対象**: `Prototypes/StickerBookThreeJS/main.js` の `BOOK_VARIANTS` に登録する book 購入者専用の表紙バリエーション
- **既存関連実装のメモ**: `BOOK_VARIANTS.book_bonus` というキーが既に存在し (`sb3d_book_bonus_cover_front_friends_armfix_20260701.webp` 等、2026-07-01 実装)、森の仲間集合をモチーフにした表紙が仮実装済。tier v3 の `book_buyer_edition` は **この既存 `book_bonus` 表紙を土台に、本ブリーフの追加要件 (右下メッセージ・名前差し込み欄) を満たすよう改修 or 差し替えるもの** であり、ゼロから無関係に作るものではない。実装時にキー名を `book_buyer_edition` にリネームするか `book_bonus` を流用するかは Phase 1 実装判断に委ねる
- **3 面の仕様**:
  1. **coverFront (表表紙)**: 絵本表紙の構図を再構成したイラスト。右下に控えめな文字「絵本を読んでくれたあなたへ」を配置 (手書き風、`#5D4E37`、主張しすぎない大きさ)
  2. **coverInside (表紙裏・見返し)**: 名前差し込み欄を持つデザイン。中央〜下部に、実装側が `localStorage` の `pono_user_name` を Canvas で動的描画するための **余白枠 (蔵書票的な罫線 or 装飾フレーム)** を確保すること。この枠内に文字を後から重ねる前提なので、枠の中には絵柄を密に描き込まない
  3. **spine (背表紙)**: 他の `BOOK_VARIANTS` 各バリエーション (boy/girl/forest/sakura/shinobi 等) の背表紙と並んだときに違和感のない太さ・配色。「book 限定」であることが一目でわかる控えめな装飾 (リボン・小さな月桂樹モチーフ等) を 1 点添える程度に留める
- **技術仕様**: 既存 `sb3d_*` テクスチャと同じ書き出し形式 (`.webp`)、既存の `boy`/`forest` バリエーションの解像度・アスペクト比に合わせる (既存ファイルを実測してから発注すること。目視で寸法が合っているか必ず確認)
- **NG**: 名前差し込み欄を絵柄で埋めてしまい後入れ文字が読めなくなる構図、既存表紙群と質感が浮くほどの過剰な装飾

## 7. 採用後のフロー

1. Claude Design で §2 の新規シール 3 種 (4 ファイル: wave base/wave up/family photo/password thanks) と §6 の特別表紙 3 面を生成
2. 生成物を Claude Code に file 渡し (mojibake 回避のため画像は直接ファイルで受け渡し、テキスト化しない)
3. Claude Code が `assets/ui/gacha/stickers/` (シール) / シール帳テクスチャ用ディレクトリ (表紙、既存 `sb3d_*` と同じ場所) に配置
4. `assets/data/game-stickers.json` に「絵本特典」8 ページ目のエントリとして `tier: book_exclusive` (または既存 `book_bonus` unlockOn タグ踏襲) で追加
5. `Prototypes/StickerBookThreeJS/main.js` の `BOOK_VARIANTS` に `book_buyer_edition` (または `book_bonus` 改修) を登録、`hydrateBookThemeCards` / `setActiveBook` の tier ゲートと接続
6. `sw.js` の `CACHE_VERSION` をバンプ (新規画像を `PRECACHE_URLS` に追加)
7. `develop-app` ブランチで push → App staging (`https://pono-asobiba-app-staging.ndw.workers.dev/`) で実機確認 → develop-app → master

## 8. 完了報告時のチェック項目 (ユーザー確認用)

Brand Kit へ依頼を出す・生成物を受け取る際、以下を確認すること。

1. **既存 book_bonus 6 枚との質感統一**: 新規 3 種を並べたときに水彩のにじみ量・線の太さ・彩度が既存と揃っているか (浮いていないか)
2. **動くシール 2 枚の一致度**: base / wave の頭身・体サイズ・カメラ距離・立ち位置が完全に一致しているか (腕の角度と表情以外は同一)
3. **縦横比の扱い**: 1024×1024 の正方形キャンバス内でモチーフが不自然に引き伸ばされていないか ([[feedback_image_aspect_ratio]] 準拠、実装側で stretch しない前提のため生成側で完結させる)
4. **家族写真シールの登場キャラ**: 絵本に実在しないキャラクターが新規デザインされていないか
5. **coverInside の名前差し込み欄**: 罫線/装飾フレーム内が空白で確保されており、後から `pono_user_name` の文字を重ねても読みやすいか
6. **解像度スケール差**: 既存 `book_bonus_*` が 512×512 なのに対し新規は 1024×1024 発注のため、実装時にダウンスケールしても線がつぶれないか (高解像度で発注し縮小する分には問題なし)
